# TC4 11.63.09 — перенос первородных стрел и дротика голема

**Цель:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Эталон:** Thaumcraft 4.2.3.5 для Minecraft 1.7.10  
**Статус:** исходники и ресурсы подготовлены; runtime и полноценная компиляция в этой среде не подтверждены.

## Реализовано

### Первородные стрелы

Шесть ранее инертных исследовательских предметов заменены на настоящие `ArrowItem`:

1. Air — тип 0;
2. Fire — тип 1;
3. Water/Frost — тип 2;
4. Earth — тип 3;
5. Order — тип 4;
6. Entropy — тип 5.

`PrimalArrowEntity` сохраняет ванильную физику стрелы и восстанавливает контракты TC4:

- базовый урон `2.1`;
- Earth: множитель `1.5`;
- Order и Entropy: множитель `0.8`;
- Fire: 5 секунд огня или 10 секунд, когда сама стрела уже горит; Enderman исключён;
- Water: Slowness на 200 тиков, усилитель 4;
- Order: Poison на 200 тиков, усилитель 4;
- Entropy: Weakness на 100 тиков;
- тип синхронизируется через `SynchedEntityData` и сохраняется в NBT;
- при подборе возвращается именно соответствующая первородная стрела;
- в полёте создаётся цветной след согласно аспекту.

Стрелы проходят через стандартный `ArrowItem#createArrow`, поэтому поддерживаются ванильным луком и существующим `BoneBowItem`.

### Дротик голема

Дротикомётчик больше не создаёт ванильный `Arrow`. Добавлен отдельный `GolemDartEntity`:

- предмет нельзя подобрать;
- сохраняется существующая формула урона голема `attackDamage × 0.4`;
- сохраняются скорость `1.6` и водная поправка точности;
- при первом клиентском тике создаются пять дымовых частиц, как у исходного `EntityDart`;
- используется отдельный renderer с уменьшенным силуэтом.

### Регистрация и сеть

Добавлены два `EntityType` через существующий `DeferredRegister`:

- `thaumcraft:primal_arrow`;
- `thaumcraft:golem_dart`.

Для первоначального появления модифицированных сущностей используется `NetworkHooks.getEntitySpawningPacket`, а меняющийся тип первородной стрелы передаётся через `SynchedEntityData`. Добавлены клиентские renderer-регистрации, английская и русская локализация.

## Ресурсы

Все шесть item-текстур первородных стрел совпадают с оригинальным банком TC4 byte-for-byte. Существующие item-модели продолжают ссылаться на:

- `el_arrow_air.png`;
- `el_arrow_fire.png`;
- `el_arrow_water.png`;
- `el_arrow_earth.png`;
- `el_arrow_order.png`;
- `el_arrow_entropy.png`.

Английские технические заглушки вида `TC4 El Arrow ...` заменены игровыми названиями.

## Неподтверждённое и отклонения

- Оригинальный renderer рисовал additive billboard из `wisp.png`; текущий безопасный перенос использует цветной particle trail. Финальная визуальная parity требует сравнения в клиенте.
- Не подтверждены фактический урон с зачарованиями, death-message attribution, подбор каждой стрелы, мультиплеерная синхронизация и dedicated-server поведение.
- Gradle Wrapper не смог загрузить Gradle 7.5.1 из-за `UnknownHostException: services.gradle.org`; `compileJava` не запускался.
- В контейнере доступна Java 21, тогда как проект рассчитан на Java 17.
