# TC4 v11.63.53 — P0 runtime-контракты критических систем

## Цель этапа

После закрытия original recipe-ledger и удаления доказанных registry-дублей следующий риск находился не в JSON, а в поведении критических систем, уже присутствующих в исходниках без обязательных GameTest-контрактов. Этап 11.63.53 добавляет пять server-only проверок и детерминированный путь миграции raw aura node.

## Перенесено и закреплено

### 1. Raw Aura Node → Node in a Jar

В `AuraNodeLegacyItem` добавлен общий метод `convertLegacyStack(ItemStack)`.

Он:

- распознаёт node payload в `TC4NodeJar`, `BlockEntityTag` или корневом `Aspects`;
- создаёт канонический `node_jar`;
- переносит полный compound узла без изменения aspect-значений;
- сохраняет custom hover name;
- возвращает `ItemStack.EMPTY`, если допустимого node payload нет.

Путь используется игровым `use()` и доступен для детерминированной world-migration/GameTest проверки.

### 2. Bone Bow

Новый GameTest фиксирует ключевой контракт:

- durability 512;
- enchantability 3;
- ускоренную charge-кривую;
- полную силу на 10 тиках;
- ограничение power;
- отдельный pull-model timing.

### 3. Traveling Trunk

Новый GameTest проверяет:

- создание entity;
- 27 слотов;
- upgrade 5;
- Forge item-handler capability;
- сохранение обычных и именованных ItemStack;
- custom entity name;
- NBT save/load round-trip.

### 4. Crimson Cultists

Новый GameTest создаёт Cultist, Knight, Cleric и Praetor и фиксирует:

- соответствие роли registry type;
- ключевые role-specific attributes;
- сохранение роли Praetor после NBT round-trip.

### 5. Thaumium Fortress Armor

Новый GameTest фиксирует:

- полный набор helmet/chest/legs;
- `goggles=1`;
- mask NBT;
- multiplier полного набора 1.30 для Grinning Devil mask;
- fire resistance всех трёх частей.

## Клиентская совместимость

В v11.63.53 добавлен guard, запрещающий прямые ссылки Java-кода проекта на внутренние mixin-классы FancyMenu, включая `ContainerWidgetPointerTracker`. Найденный ранее краш инвентаря не воспроизводится как статическая зависимость исходников порта. Полноценный client smoke-test с FancyMenu всё ещё нужен на собранном JAR.

## Runtime-статус

Точный JAR v11.63.53 не получен. После восстановления `gradle-wrapper.jar` wrapper остановился при загрузке Gradle 7.5.1:

```text
java.net.UnknownHostException: services.gradle.org
GRADLE_RC=1
```

В окружении установлен OpenJDK 21.0.10, а целевой Forge 1.19.2 проект должен собираться и тестироваться на Java 17. Поэтому 24 GameTests имеют честный статус **NOT_TESTED**. Подтверждённый результат v11.63.51 — 18/18 PASS — остаётся только предыдущим baseline.
