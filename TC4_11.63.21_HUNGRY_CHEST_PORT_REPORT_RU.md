# Thaumcraft Legacy Rebuild 1.19.2 — v11.63.21

## Стадия

**Hungry Chest / Голодный сундук** — завершение серверной логики поглощения предметов, единого инвентарного контракта Forge и оригинальной анимации крышки.

Целевая платформа: Minecraft 1.19.2, Forge 43.5.2, Java 17.  
Эталон поведения: Thaumcraft 4.2.3.5 для Minecraft 1.7.10.

## Исходный контракт TC4

По оригинальным `BlockChestHungry`, `TileChestHungry`, `BlockChestHungryRenderer` и `TileChestHungryRenderer` восстановлены следующие свойства:

- горизонтально ориентированный сундук с формой, утопленной на 1/16 по X/Z и высотой 14/16;
- обычный трёхрядный инвентарь на 27 доступных слотов;
- поглощение столкнувшихся предметных сущностей через тот же инвентарь;
- сохранение остатка, если помещается только часть стака;
- звук поедания `random.eat` с громкостью `0.25` и высотой `(random - random) * 0.2 + 1.0`;
- событие блока `(2, 2)`, поднимающее крышку как минимум до `0.2`;
- движение крышки с шагом `0.1` за тик;
- звук открытия только из полностью закрытого состояния;
- звук закрытия при пересечении угла `0.5` вниз;
- периодическая коррекция числа открывших игроков;
- исходная геометрия ModelChest, UV и текстура `chesthungry.png`.

## Реализовано

### Единый инвентарный путь

- `HungryChestBlockEntity` сохраняет один `InvWrapper`, опубликованный как `ForgeCapabilities.ITEM_HANDLER` со всех сторон.
- Поглощение предметов теперь выполняется через `ItemHandlerHelper.insertItemStacked`.
- Автоматизация, меню и предметы в мире используют один 27-слотовый контейнер как источник истины.
- Сначала заполняются совместимые существующие стаки, затем свободные слоты.
- NBT предмета не теряется: вставляется копия исходного `ItemStack`.
- При частичной вставке предметная сущность получает точный остаток.
- При полном отказе исходный стак остаётся без изменений, а звук и анимация не запускаются.
- При полном поглощении предметная сущность удаляется.
- Любое успешное изменение обновляет comparator-сигнал соседей.

### Звук и анимация

- Восстановлен звук `minecraft:entity.generic.eat` с громкостью `0.25`.
- Восстановлена треугольная формула высоты `(r1 - r2) * 0.2 + 1.0`.
- Успешное поглощение посылает событие блока `(2, 2)` и поднимает крышку минимум до `0.2`.
- `lidAngle` движется к целевому состоянию с шагом `0.1` за тик и ограничивается диапазоном `0..1`.
- Звук открытия воспроизводится только при переходе из полностью закрытого состояния.
- Звук закрытия воспроизводится при пересечении угла `0.5` вниз.
- Высота звуков открытия и закрытия: `random * 0.1 + 0.9`.
- Клиентский renderer продолжает использовать кубическое сглаживание угла крышки.

### Открыватели и меню

- Открытие использует штатный `ChestMenu.threeRows`.
- `startOpen` и `stopOpen` синхронизируют число открывших игроков через block event `1`.
- Сервер каждые 80 тиков пересчитывает игроков поблизости, чьё активное меню действительно связано с данным контейнером.
- Это предотвращает зависание крышки после разрыва соединения или аварийного закрытия меню.

### Сохранённые контракты

- форма `14 × 14 × 14` пикселей с отступом 1 пиксель по горизонтали;
- четыре горизонтальных направления;
- comparator output;
- выпадение содержимого при разрушении;
- item-handler capability со всех сторон;
- `ENTITYBLOCK_ANIMATED` и существующий BER/BEWLR;
- исходная ModelChest-геометрия и UV;
- три копии `chesthungry.png` подтверждены как byte-exact относительно сохранённого оригинала.

## Компиляция

Запущено:

```text
./gradlew compileJava --stacktrace --no-daemon
```

Gradle Wrapper остановился до запуска Java-компилятора:

```text
Downloading https://services.gradle.org/distributions/gradle-7.5.1-bin.zip
java.net.UnknownHostException: services.gradle.org
```

В контейнере доступен OpenJDK 21.0.10. Отдельный JDK 17 не найден. Поэтому:

- JAR не создан;
- client и dedicated server не запускались;
- столкновение предметов, hopper-вставка и частичные остатки не проверены в игре;
- два одновременно открывших игрока не проверены;
- звук, анимация, ориентации и item transforms не проверены визуально;
- save/reload и resource reload имеют статус `NOT_TESTED`.

## Известные адаптации

- Старый `InventoryUtils.placeItemStackIntoInventory` заменён публичным Forge item-handler API.
- Периодическая проверка открывших игроков адаптирована через `ChestMenu.getContainer()` вместо старого GUI/container-класса.
- Старый числовой TESR и ручной item renderer остаются уже перенесёнными современными BER/BEWLR.
- Текстура и модель не перерабатывались художественно: сохранён исходный внешний контракт.
