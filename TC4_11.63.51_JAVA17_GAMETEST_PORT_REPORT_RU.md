# TC4 v11.63.51 — Java 17 build и первый полный Forge GameTest

**Дата:** 2026-07-19

## Найденные ошибки v11.63.50

Реальная компиляция Forge 1.19.2 обнаружила четыре ошибки, которые не замечали текстовые guard-проверки:

- `FumeDissipatorBlock.canSurvive` принимал `LevelAccessor` вместо сигнатуры `LevelReader`;
- `ArcaneDoorItem` дважды передавал `UseOnContext` в `BlockState.canBeReplaced`;
- `TC4ManaBeanItem` делал тот же неправильный вызов при посадке Mana Pod.

В обоих item-классах теперь создаётся штатный `BlockPlaceContext`, поэтому проверка заменяемости блока соответствует API Forge/Minecraft 1.19.2.

## Runtime GameTest

Первый запуск выполнил 18 тестов и выявил одно падение `energizedNodeUsesTransientPerTickPool`. Производственный пул vis обновлялся по времени мира правильно, но тест загружал противоречивое состояние: `Energized=true` при `EnergizedTicks=0`. На следующем серверном тике узел закономерно выключался до проверки пополнения.

Все три energized-node fixture теперь записывают `EnergizedTicks=100`, как нормальный сохранённый активный узел. Повторный полный запуск завершился:

- **18/18 required GameTests PASS**;
- Gradle task `runGameTestServer`: **BUILD SUCCESSFUL**;
- Magical Forest присутствует в активном Overworld `MultiNoiseBiomeSource`;
- загружено 70 алхимических, 114 мистических и 78 инфузионных рецептов;
- Outer Lands успешно создаётся и сохраняется сервером.

## Граница доказательства

Это реальный server-side runtime PASS обязательного GameTest-пакета. Он не доказывает визуальный паритет клиента, GUI/JEI, multiplayer с двумя клиентами и миграцию старых миров — эти направления остаются следующими этапами.
