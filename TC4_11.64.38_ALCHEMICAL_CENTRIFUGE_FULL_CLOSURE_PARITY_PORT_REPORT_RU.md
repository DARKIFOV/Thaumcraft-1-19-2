# TC4 v11.64.38 — Alchemical Centrifuge: полный source/resource closure

## Единственная цель раунда

В этом релизе повторно проверялась и закрывалась только **Alchemical Centrifuge / Алхимическая центрифуга**:

- приём составной essentia снизу;
- прямое подключение трубы, банки, алембика и другой центрифуги;
- 39-тиковый цикл разделения;
- случайный выбор одного из двух непосредственных компонентов;
- suction и направления транспорта;
- redstone pause;
- сохранение и сетевой sync;
- вращение, звук, модель и item-render;
- форма, прочность и звук блока;
- рецепт и исследование `CENTRIFUGE`.

Thaumatorium и Essentia Crystalizer в этот раунд не включались.

Точные SHA-256, выражения и номера строк записаны в `TC4_11.64.38_ALCHEMICAL_CENTRIFUGE_SOURCE_EVIDENCE.json`.

## Что было неверно до v11.64.38

- Блок сохранял и загружал временные `process`, `counter`, `rotation` и `rotationSpeed`, которых нет в оригинальном NBT.
- Не сохранялся исходный `facing`.
- Подача была фактически привязана к трубе и не воспроизводила generic connectable path оригинала для прямой банки, алембика и каскада центрифуг.
- Блок имел неправильные hardness/resistance и полную кубическую форму.
- Правый клик выводил отладочное сообщение.
- Renderer показывал только вращаемую часть, а статическая геометрия подменялась generic block model.
- Item renderer вращал механизм, хотя оригинальный inventory render использует новую TileCentrifuge с нулевым вращением.
- Сетевой update packet вызывал полный `load()` и сбрасывал клиентскую скорость/угол при каждом изменении содержимого.
- В рецепте использовался Thaumatorium вместо исходной Alchemical Construct.
- Доказательный набор не охватывал прямую банку, прямой алембик, каскад центрифуг, canonical NBT и точную форму.

## Production-исправления

### Серверная логика

- Suction снизу — **64**, сверху и по горизонтали — 0.
- Подключение только UP/DOWN; вход только DOWN, выход только UP.
- Минимальный suction — 0.
- Принимается только один непримальный аспект.
- После приёма устанавливается `process = 39`.
- Попытка забора выполняется каждые 5 тиков только при пустом input/output.
- Redstone полностью приостанавливает забор и countdown.
- При завершении выбирается один из двух прямых компонентов через world RNG `nextInt(2)`.
- Output capacity — ровно одна essentia.
- Поддержаны прямые источники под блоком:
  - `EssentiaTubeBlockEntity`;
  - `EssentiaJarBlockEntity`;
  - `AlembicBlockEntity`;
  - `AlchemicalCentrifugeBlockEntity`.
- Каскад центрифуг может последовательно раскладывать сложные аспекты.

### NBT и сеть

Установленная центрифуга сохраняет только:

- `aspectIn`;
- `aspectOut`;
- `facing`.

Не сохраняются:

- `process`;
- `counter`;
- `rotation`;
- `rotationSpeed`.

При загрузке мира временные поля сбрасываются. При update packet и update tag обновляются только три исходных persistent-поля, поэтому клиентское вращение не обрывается.

### Client и ресурсы

- Скорость увеличивается на 2 до максимума 20.
- При остановке скорость уменьшается на 0,5.
- Звук `pump` воспроизводится на исходном пересечении половины оборота только локально на клиенте.
- Модель восстановлена из шести исходных частей:
  - Crossbar;
  - Dingus1;
  - Dingus2;
  - Core;
  - Top;
  - Bottom.
- Верх и низ остаются неподвижными; вращается только внутренняя сборка.
- Item renderer статичен.
- `centrifuge.png` побайтово совпадает с оригиналом.
- Block JSON оставлен как BER-only carrier с корректной particle texture.

### Блок, рецепт и исследование

- Shape/collision: `4..12 × 0..16 × 4..12`, то есть исходный столб 8×16×8.
- Hardness — 0,5.
- Resistance — 5.
- Sound — metal.
- Arcane recipe:
  - ` T `;
  - `ACP`;
  - ` T `;
  - T — обычная essentia tube;
  - A — Alembic;
  - C — **Alchemical Construct**;
  - P — piston;
  - 5 Aqua, 5 Ordo, 5 Perditio.
- Исследование `CENTRIFUGE`:
  - категория ALCHEMY;
  - координаты 10,0;
  - complexity 2;
  - Perditio/Praecantatio/Permutatio/Fabrico по 3;
  - parent `TUBEFILTER`;
  - concealed;
  - страницы Centrifuge и TubeBuffer сохранены.
