## 2. Thaumatorium

Старая реализация считала трубную сеть единым плоским хранилищем и могла мгновенно списать полную стоимость рецепта. Это принципиально отличалось от TC4.

Теперь Thaumatorium:

- хранит катализатор отдельно;
- хранит уже полученную эссенцию внутри Block Entity;
- определяет выбранную алхимическую формулу;
- ищет первый недостающий аспект;
- выставляет suction `128` только для этого аспекта;
- принимает ровно одну единицу эссенции;
- выполняет попытку втягивания раз в 5 тиков;
- не забирает сразу полный набор из всей сети;
- останавливается по красному камню;
- требует источник нагрева двумя блоками ниже;
- поддерживает огонь, огонь душ, лаву и Nitor;
- не поглощает эссенцию при забитом выходном инвентаре;
- сохраняет внутренний буфер, формулу и катализатор в NBT;
- синхронизирует состояние клиенту.

### 2.1 Порты конструкции

Трубный транспорт видит оба уровня алхимической конструкции. Верхняя часть не имеет отдельного Block Entity в современной адаптации, поэтому соединение разрешается обратно к нижнему контроллеру.

Не подключаются:

- нижняя сторона;
- лицевая сторона выхода готового предмета.

### 2.2 Создание результата

Перед завершением рецепта выполняется предварительная проверка результата и выходного инвентаря.

Транзакция:

1. проверяет накопленную эссенцию;
2. проверяет катализатор;
3. проверяет выход;
4. списывает аспекты;
5. списывает один катализатор;
6. создаёт результат;
7. вставляет его в соседний Forge Item Handler или ванильный Container;
8. при отсутствии инвентаря выбрасывает результат с лицевой стороны;
9. воспроизводит звук и частицы.

### 2.3 Mnemonic Matrix

Базовая вместимость формул:

```text
1 формула
```

Каждая соседняя матрица добавляет:

```text
+2 формулы
```

Используется формула:

```text
1 + matrices × 2
```

Матрицы проверяются возле обоих уровней конструкции.

## 3. Alchemical Centrifuge

Добавлены функциональные:

```text
AlchemicalCentrifugeBlock
AlchemicalCentrifugeBlockEntity
AlchemicalCentrifugeRenderer
```

### 3.1 Транспорт

Центрифуга:

- соединяется только сверху и снизу;
- принимает эссенцию только снизу;
- отдаёт результат только сверху;
- выставляет suction `64` снизу;
- не принимает первичные аспекты;
- забирает одну единицу раз в 5 тиков;
- останавливается от сигнала красного камня.

### 3.2 Разделение

После принятия составного аспекта:

```text
process = 39 тиков
```

После завершения случайно выбирается один из двух компонентов аспекта. Результат хранится как одна единица до забора трубой, буфером или эссенционным големом.

### 3.3 Рендер

Перенесена геометрия вращающейся части `ModelCentrifuge`:

- Crossbar;
- Dingus1;
- Dingus2;
- Core.

Статические верхний и нижний корпуса остаются в block model. Ротор вращается отдельным современным Block Entity renderer.

Восстановлены:

- ускорение вращения на `+2`;
- максимальная скорость `20`;
- замедление на `0,5`;
- исходная периодичность звука `pump` при прохождении половины оборота;
- оригинальная текстура `textures/models/centrifuge.png`.

## 4. Обычная алхимическая печь

Печь полностью переведена с прямого клика-буфера на два настоящих слота:

```text
0 — сырьё
1 — топливо
```

### 4.1 Sided inventory

Современный Block Entity реализует `WorldlyContainer`.

Стороны:

```text
верх: предметы не принимаются
боковые стороны: сырьё
низ: топливо
```

Это возвращает работу с:

- воронками;
- ванильной автоматизацией;
- адаптерами модовых инвентарей;
- корректным сохранением стаков.

### 4.2 Вместимость

Обычная печь:

```text
50 essentia
```

Современный совместимый блок advanced-версии:

```text
500 essentia
```

Полный многоблочный контроллер advanced-печи остаётся отдельным этапом и не считается законченным в этой версии.

### 4.3 Время переработки

Формула TC4:

```text
visSize × 10 × (1 − 0,125 × bellows)
```

Поддерживается до четырёх мехов.

Печь:

- получает аспекты предмета через общую Aspect Database;
- проверяет свободное место до начала;
- не расходует предмет при переполнении;
- сохраняет прогресс, топливо, сырьё и аспекты;
- сбрасывает прогресс при извлечении сырья.

### 4.4 Alumentum

Исправлено отдельное значение топлива из `EventHandlerWorld` оригинального TC4:

```text
6400 тиков
```

Alumentum также включает ускоренное распределение эссенции в алембики:

```text
20 тиков вместо 40
```

Обычное ванильное и Forge-топливо определяется через `ForgeHooks.getBurnTime`.

## 6. Распределение печь → алембики

Обычная печь больше не считается прямым источником труб.

Это важная правка: в TC4 её эссенция сначала распределяется только в вертикальную колонну алембиков.

Поддерживается до пяти последовательных алембиков.

Алгоритм:

1. раз в 40 тиков, либо 20 с Alumentum;
2. сначала каждый уже занятый алембик получает одну единицу своего аспекта;
3. обслуженные аспекты временно исключаются;
4. затем первый пустой алембик получает аспект своего фильтра;
5. без фильтра выбирается случайный ещё не обслуженный аспект;
6. за один проход в пустой алембик помещается только одна единица.
