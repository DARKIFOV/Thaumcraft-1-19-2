# Thaumcraft Legacy Rebuild 11.62.85 — runtime initialization hotfix

**Дата:** 2026-07-15  
**Платформа проекта:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Источник диагностики:** `crash-2026-07-15_18.22.12-client.txt`  
**Объективный статус:** `STATIC PASS / CLIENT INITIALIZATION REQUIRES RERUN / RUNTIME NOT PASSED`

## 1. Результат crash-report

JAR 11.62.84 успешно дошёл до реальной регистрации предметов Forge. Это подтверждает, что предыдущие compile-hotfix исправили показанные compiler-блокеры. Инициализация остановилась на двух независимых ошибках.

### 1.1. Thaumcraft: Bone Bow

Корневая ошибка:

```text
java.lang.RuntimeException: Unable to have damage AND stack.
at com.darkifov.thaumcraft.item.BoneBowItem.<init>(BoneBowItem.java:27)
```

В 11.62.84 конструктор содержал:

```java
super(properties.durability(512).stacksTo(1));
```

Для Minecraft 1.19.2 повреждаемый предмет уже имеет размер стека 1. Дополнительный вызов `stacksTo(1)` после `durability(512)` запрещён самим `Item.Properties` и выбрасывает исключение во время регистрации.

Исправление 11.62.85:

```java
super(properties.durability(512));
```

Игровая логика Bone Bow не менялась: сохранены durability 512, ускоренная TC4-кривая натяжения, скорость стрелы, базовый бонус урона, enchantments, расход стрел и model predicates.

### 1.2. Vampirism

Crash-report одновременно содержит отдельную ошибку Vampirism 1.9.5:

```text
java.lang.NullPointerException: Registry Object not present: vampirism:garlic_bread
at de.teamlapen.vampirism.entity.minion.management.MinionTasks.lambda$static$1(MinionTasks.java:34)
```

Она не вызвана Thaumcraft. После исправления Bone Bow именно Vampirism может стать следующим блокером запуска. Его нужно проверять отдельно на чистом профиле с корректными зависимостями и конфигурацией.

## 2. Защита от повторения

Добавлен `tc4_116285_runtime_init_hotfix_guard.py`, который:

- проверяет точный безопасный конструктор Bone Bow;
- запрещает прежнюю цепочку `durability(512).stacksTo(1)`;
- сканирует все Java-исходники на цепочки `durability(...).stacksTo(...)` и обратный порядок;
- проверяет версии source/workflow/runtime manifest;
- запускается в обоих GitHub Actions workflow до Gradle build.
