# Thaumcraft Legacy Rebuild 11.62.93 — Wand variants, vis, JEI and texture hotfix

**Дата:** 2026-07-16  
**Платформа:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**База:** 11.62.92 Full Texture/UV Audit Hotfix  
**Эталон поведения:** Thaumcraft 4.2.3.5

## 1. Диагноз по предоставленным скриншотам и исходникам

Порт хранит обычные жезлы, посохи и скипетр как варианты одного registry-item. Их личность задаётся корневыми NBT-тегами `rod`, `cap` и `sceptre`. При отсутствии этих тегов `WandComponentData` закономерно использует fallback `wood + iron`, то есть ёмкость 25 vis и соответствующие текстуры.

В 11.62.92 кодовые пути расходились:

- реальный generated Arcane Workbench output умел записывать rod/cap NBT;
- `ArcaneWorkbenchRecipe.result()` оставался общим нетегированным предметом;
- JEI показывал именно общий `recipe.result()`, поэтому разные рецепты визуально схлопывались в wood/iron;
- творческая вкладка содержала только три исторических registry alias вместо полного NBT-каталога;
- JEI не имел subtype interpreter для rod/cap/sceptre.

Это объясняет наблюдаемые 25 vis и одинаковые текстуры в тех контекстах, где терялся компонентный NBT. Сами активные PNG не были причиной: все 16 уникальных rod/cap-текстур, используемых новыми вариантами, совпали с сохранёнными оригинальными TC4-файлами побайтово.

## 2. Исправления 11.62.93

### 2.1. Единый авторитетный результат рецепта

Добавлен `WandCraftingRuntime.resultFor(ArcaneWorkbenchRecipe)`. Он извлекает rod/cap из metadata generated-рецепта и создаёт NBT-результат через `WandVariantRuntime.create(...)`. Тот же метод теперь используют реальный Arcane Workbench и JEI.

Результат крафта остаётся пустым по vis, как игровой предмет, но получает правильную максимальную ёмкость из rod NBT.

### 2.2. Полный творческий каталог

Добавлен `WandVariantRuntime`:

- 18 craftable rod-вариантов;
- 6 craftable cap-вариантов;
- 108 комбинаций жезлов/посохов;
- один оригинальный canonical silverwood/thaumium sceptre;
- всего **109 NBT-вариантов**.

Creative-only rod и infinity cap не дублируются: отдельный Avaritia creative wand сохранён. Исторические registry IDs `iron_capped_wooden_wand`, `greatwood_wand`, `silverwood_wand` остаются зарегистрированы для совместимости миров, но их дубли удаляются из творческой выдачи.

Как в оригинальном `ItemWandCasting#getSubItems`, creative-варианты заполнены до собственной максимальной ёмкости.

### 2.3. JEI

`TC4JeiPlugin.registerItemSubtypes` регистрирует component-only subtype interpreter:

```java
registration.registerSubtypeInterpreter(
    ThaumcraftMod.IRON_CAPPED_WOODEN_WAND.get(),
    (stack, context) -> WandVariantRuntime.subtypeKey(stack)
);
```

Ключ включает только rod/cap/sceptre. Текущий заряд vis и focus NBT намеренно игнорируются: заряженный creative stack и пустой recipe output остаются одной визуальной разновидностью, но разные материалы больше не схлопываются.

JEI Arcane category показывает динамический NBT-output generated-рецепта вместо общего fallback stack.

### 2.4. Ёмкость vis

Ёмкость вычисляется из компонентов стека:

- wood: 25;
- greatwood: 50;
- elemental rods: 75;
- silverwood: 100;
- staff rods: 125/175/250;
- silverwood sceptre: 150.

Внутреннее хранение остаётся в centivis. Максимум не зависит от cap; cap меняет стоимость расхода vis.

### 2.5. Текстуры

`WandItemRenderer` получает `WandComponentData` из каждого ItemStack и выбирает `rod.rendererTexture()` и `cap.rendererTexture()`. После восстановления NBT identity это даёт правильную пару оригинальных текстур в GUI, JEI, ground, fixed, first-person и third-person путях BEWLR.

PNG не перерисовывались: 16/16 используемых уникальных wand component textures — byte-exact original.
