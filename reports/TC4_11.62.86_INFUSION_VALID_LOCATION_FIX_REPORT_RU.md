# Thaumcraft Legacy Rebuild 11.62.86 — исправление Infusion Matrix

**Дата:** 2026-07-15  
**Платформа:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Основа:** 11.62.85 Runtime Init Hotfix

## Исправленный дефект

В 11.62.85 `InfusionAltarStructure.valid` ошибочно зависел от наличия хотя бы одного непустого бокового пьедестала. Во время наполнения компоненты снимаются с пьедесталов по одному. После снятия последнего компонента структура становилась «невалидной», хотя центральный пьедестал и четыре колонны оставались на месте. Проверка в `tickServer()` или `finishInfusion()` могла сорвать корректный рецепт перед завершением.

В оригинальном `TileInfusionMatrix.validLocation()` (строки 234–246 декомпилированного TC4 4.2.3.5) проверяются только:

1. центральный `TilePedestal` на два блока ниже матрицы;
2. четыре `TileInfusionPillar` по диагоналям на той же высоте.

Количество оставшихся компонентов в `validLocation()` не участвует.

## Изменения 11.62.86

- структурная валидность теперь равна `strictTc4Location`;
- наличие компонентов проверяется отдельно только при старте рецепта;
- уже запущенный рецепт не ломается после штатного снятия последнего компонента;
- обычные `arcane_stone_bricks` больше не подменяют специальные infusion pillars;
- для валидной структуры требуются четыре блока `thaumcraft:infusion_pillar`;
- добавлен `tc4_116286_infusion_valid_location_guard.py`;
- guard подключён к build/release workflows.
