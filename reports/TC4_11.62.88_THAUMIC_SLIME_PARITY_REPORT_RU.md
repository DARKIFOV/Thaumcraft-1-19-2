# Thaumcraft Legacy Rebuild 11.62.88 — сравнение с TC4 4.2.3.5 и перенос Thaumic Slime

**Дата:** 2026-07-16  
**Цель:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Эталон:** Thaumcraft 4.2.3.5 для Minecraft 1.7.10  
**Статус:** `STATIC SOURCE CONTRACT PASS / BUILD REQUIRES CI / RUNTIME NOT TESTED`

## 1. Источники сравнения

- официальный файл Thaumcraft 4.2.3.5: `https://www.curseforge.com/minecraft/mc-mods/thaumcraft/files/2227552`;
- `ConfigEntities.java` и `EntityThaumicSlime.java` из исходного дерева TC4 4.2.3.5, доступного в общественном порте `0FL01/Thaumcraft-4.2-FOREVA`;
- отчёт 11.62.86 по `InfusionAltarStructure.validLocation()`;
- отчёт и скомпилированный JAR 11.62.87 по Wisp;
- исходное дерево 11.62.85 как последняя предоставленная исходная база.

### Ограничение исходной базы

Загруженный архив `thaumcraft-legacy-rebuild-v11.62.87.zip` содержит JAR и отчёт, но не Java-исходники. Поэтому 11.62.88 является **восстановленным исходным деревом**:

1. исходники 11.62.85;
2. документированная правка Infusion Matrix 11.62.86;
3. ресурсы и восстановленные source-контракты Wisp из JAR/отчёта 11.62.87;
4. новый перенос Thaumic Slime.

Это не побайтовое восстановление исходников, из которых был собран JAR 11.62.87. Функциональные контракты 11.62.86–11.62.87 защищены отдельными guards.

## 2. Реестр сущностей: оригинал и порт

Оригинальный `ConfigEntities.init()` регистрирует **50 типов сущностей**. После добавления Thaumic Slime проект 11.62.88 регистрирует **30 типов**.

Разница `30/50` не означает ровно 20 отсутствующих мобов: оригинальные 50 включают специальные ItemEntity, рыболовный поплавок и отдельные снаряды, часть которых в 1.19.2 заменена общей реализацией focus-projectile. Тем не менее перенос реестра ещё не завершён.

Сохранены из предыдущих версий:

- Infusion Matrix valid-location parity 11.62.86;
- Wisp parity 11.62.87, включая собственный аспект в NBT, Wisp Essence, renderer, Nether spawn и динамический Thaumometer scan;
- compile/runtime hotfixes 11.62.83–11.62.85.

### Адаптация к современному API

Оригинал вручную управлял прыжками в `updateEntityActionState`. Порт наследует современный `Slime` jump/move controller, а TC4-специфичные spit/merge/size/split правила выполняет серверная логика `TC4ThaumicSlimeEntity`. После вызова modern `Slime#setSize` скорость принудительно возвращается к 0.20, чтобы крупная слизь не получила неоригинальное линейное ускорение vanilla Slime.

## 7. Что не подтверждено

Локальный Gradle wrapper не смог разрешить `services.gradle.org` (`UnknownHostException`). Поэтому для 11.62.88 не заявляются:

- `compileJava PASS`;
- успешный `gradlew build`;
- JAR;
- SHA-256 JAR;
- client/dedicated-server runtime PASS;
- визуальная и поведенческая parity Thaumic Slime в игре.

## 8. Обязательный runtime-протокол Thaumic Slime

1. Spawn egg создаёт размеры 1/2/4 и оригинальные цвета яйца.
2. Размеры 4+ плюются маленькой слизью по игроку дальше четырёх блоков и уменьшаются на 1.
3. Два слизня объединяются после 100 тиков, но не превышают размер 100.
4. Размеры 2–3 не создают потомка при смерти; размер 4 создаёт 2; размер 9 создаёт 3.
5. Урон/здоровье/XP соответствуют размеру.
6. Save/reload сохраняет `Size`, spit cooldown и launched state.
7. Маленькие слизни дают Tainted Goo примерно в 1/3 смертей.
8. Клиент и dedicated server не получают рассинхронизацию размеров/renderer.
