# Thaumcraft Legacy Rebuild 11.62.77 — исправление цикла Outer Lands

**Дата:** 2026-07-14  
**Платформа:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Эталон:** Thaumcraft 4.2.3.5 (`WorldProviderOuter`, `ChunkProviderOuter`, `TeleporterThaumcraft`, `GenPortal`, `GenBossRoom`, `TileEldritchLock`)  
**Статус версии:** **FAIL / НЕ ГОТОВА К РЕЛИЗУ**  
**Статус Outer Lands:** **STATIC PARTIAL / RUNTIME NOT TESTED**

## 1. Цель итерации

Версия 11.62.76 содержала регистрацию измерения, генераторы комнат, телепортер, `SavedData`, блок Eldritch Lock и несколько сущностей. Наличие этих файлов ошибочно могло восприниматься как готовая система. Цель 11.62.77 — исправить проверяемые архитектурные расхождения в последовательности:

`вход через портал → создание единой сети лабиринта → генерация комнаты входа → генерация boss-room с lock → активация lock Eldritch Eye → задержка 100 тиков → появление босса в центре комнаты`.

Runtime PASS не заявляется: клиент, выделенный сервер, сохранение/загрузка и возврат в Overworld не запускались.

## 2. Подтверждённые ошибки 11.62.76

| Ошибка | Поведение 11.62.76 | Эталон TC4 | Риск |
|---|---|---|---|
| Координаты входной комнаты | произвольные координаты исходного портала использовались как угол комнаты | `GenPortal` получает координаты ячейки 16×16 | перекрывающиеся комнаты и смещённый лабиринт |
| Второй портал | после генерации создавался дополнительный портал в углу | портал комнаты находится в локальной точке `(8,3,8)` | игрок попадал не в фактический портал комнаты |

## 3. Выполненные изменения

### 3.1. Вход и телепортация

`TC4OuterLandsTeleporter` теперь:

- выравнивает X/Z входа по сетке ячеек 16×16 через `Math.floorDiv`;
- создаёт maze только из пути входного портала;
- использует портал, созданный комнатой в локальной позиции `(8,3,8)`;
- удаляет безопасное пространство рядом с реальным порталом;
- больше не создаёт вторую платформу 5×5 и второй угловой портал;
- сохраняет исходную точку возврата игрока через `PlayerThaumData`.

### 3.2. Геометрия GenPortal

`TC4OuterLandsGenCommonAdapter.generatePortalRoom` дополнен элементами оригинального `GenPortal`:

- ступенчатый нижний пол;
- зеркальный ступенчатый потолок;
- четыре полые угловые шахты;
- нижние и верхние группы лестниц;
- центральный altar/portal/obelisk остаются в координатах TC4.

Это source-level перенос геометрии. Визуальное совпадение должно быть подтверждено side-by-side скриншотами.

### 3.3. Разделение создания maze и population

`TC4OuterLandsChunkProviderBridge.populateLikeTC4`:

- больше не вызывает `ensurePortalMaze`;
- больше не вызывает `generateAround`;
- обрабатывает только ячейки, уже существующие в `TC4OuterLandsMazeHandler`;
- возвращает `boolean`, чтобы вызывающий код не помечал несуществующий чанк как сгенерированный.

`TC4OuterLandsLivePopulateAdapter` записывает чанк в `SavedData` только после успешной генерации существующей maze-ячейки.

### 3.4. Boss-room и Eldritch Lock

`TC4OuterLandsBossRoomPlacer.placeBossRoomCell` теперь:

- создаёт 2×2 boss-room;
- размещает `ELDRITCH_LOCK` для palette code 16;
- не создаёт Warden или Golem во время worldgen.

`EldritchLockBlock`:

- принимает только `ELDRITCH_EYE`;
- расходует предмет только на сервере;
- запускает существующую 100-тиковую последовательность BlockEntity;
- воспроизводит звук активации runic shield.

### 3.5. Поиск комнаты и появление босса

`TC4EldritchLockBossSpawner`:

- просматривает 5×5 maze-ячеек вокруг lock;
- использует feature 2 как северо-западный anchor 2×2 комнаты;
- использует features 2–5 и соединения для выбора четверти Warden;
- вычисляет геометрический центр 2×2 boss-room;
- привязывает созданного моба к центру комнаты через home restriction и NBT breadcrumbs;
- сохраняет последовательность вариантов, зависящую от `bossCount % 4`.
