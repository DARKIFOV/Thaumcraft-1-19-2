# Thaumcraft Legacy Rebuild 11.62.81 — Warp/Eldritch parity

**Дата:** 2026-07-15  
**Платформа:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Эталон:** Thaumcraft 4.2.3.5  
**Статус:** `STATIC SOURCE/NETWORK CONTRACT PASS; BUILD FAIL; RUNTIME NOT TESTED`

## 1. Область итерации

Версия 11.62.81 исправляет подтверждённые расхождения в хранении и синхронизации warp, работе Warp Ward, редких WarpEvents, поиске позиции для Eldritch Guardian/Mind Spider и геометрии Death Gaze. Изменения сверялись с оригинальными классами TC4 `WarpEvents`, `PlayerKnowledge` и связанными предметами/эффектами.

Наличие кода не считается доказательством gameplay parity. Все runtime-статусы остаются `NOT TESTED`, пока отсутствуют SHA-256-проверенные клиентские/серверные артефакты.

## 2. Исправления

### 2.1. Раздельные значения warp и сетевой пакет

Ранее клиент получал только суммарное значение warp. В 11.62.81 `ResearchSyncPacket` передаёт:

- суммарный warp;
- permanent warp;
- sticky warp;
- temporary warp;
- WarpCounter.

`ClientResearchData` хранит все пять значений отдельно. Это устраняет потерю состояния на клиенте и создаёт корректную основу для GUI, диагностических overlay и multiplayer-проверок.

### 2.2. Warp Ward: один авторитетный таймер

Ранее Warp Ward одновременно существовал как `MobEffect` и как отдельный persistent NBT-таймер. Из-за двух источников истины эффект мог продолжать блокировать WarpEvents после удаления молоком или командой.

В 11.62.81:

- runtime-источником состояния является только `WARP_WARD` MobEffect;
- старый `WarpWardTicks` считывается и удаляется один раз;
- при загрузке старого мира значение мигрируется в реальный MobEffect;
- Purifying Fluid и Warp Ward Talisman больше не создают второй NBT-таймер;
- Sanity Soap проверяет реальный эффект, а не устаревшее NBT-состояние.

### 2.3. Редкое очищение sticky warp

Оригинальный редкий WarpEvent уменьшает sticky warp напрямую и не сбрасывает счётчик событий. Публичный метод порта `addWarpSticky` также пересчитывал WarpCounter, поэтому использовался неверный путь.

Добавлен отдельный `decayStickyWarpFromEvent`, который:

- уменьшает только sticky warp;
- не меняет permanent/temporary warp;
- не сбрасывает WarpCounter.

### 2.4. Eldritch Guardian и Mind Spider

Ранее при неудаче поиска безопасной позиции сущность принудительно появлялась возле игрока. Это могло создавать её внутри блока, жидкости или в невалидной позиции.

Новый поиск:

- выполняет до 50 попыток;
- использует исходные знаковые смещения по X/Y/Z в диапазоне 7–24;
- проверяет границы высоты мира;
- требует прочную поверхность снизу;
- требует два свободных воздушных блока;
- запрещает жидкости;
- проверяет collision box конкретной сущности;
- отменяет событие, если подходящей позиции нет.

### 2.5. Death Gaze

Ранее эффект применялся по окружающей области без полноценной фронтальной проверки. В 11.62.81 восстановлен направленный конус:

- нормализованное скалярное произведение направления взгляда и направления на цель должно быть не меньше `0.75`;
- сохраняется проверка line of sight;
- учитывается серверная настройка PvP;
- цель получает `lastHurtByPlayer`;
- Mob начинает атаковать игрока;
- уже поражённые Wither цели пропускаются.
