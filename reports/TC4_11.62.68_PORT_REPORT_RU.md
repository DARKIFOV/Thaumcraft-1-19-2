# Thaumcraft Legacy Rebuild 11.62.68 — отчёт о продолжении портирования

## Итог

Итерация 11.62.68 продолжает блок Warp, добавленный в 11.62.67, и переносит связанную цепочку **Bath Salts → Purifying Fluid → Warp Ward → Arcane Spa**. Ранее предметы и текстуры присутствовали среди исследовательских зеркал, а рецепты существовали в data-файлах, но у них не было полноценной жидкости, размещаемого источника и исполняемой Spa-механики.

Статус выпуска: **PARTIAL / STATIC PASS**. Это исходный проект для дальнейшей сборки и runtime-проверки, а не подтверждённый релизный JAR.

## 1. Purifying Fluid

Добавлена полноценная Forge-жидкость:

- `FluidType` с исходными характеристиками: light 10, density 1000, viscosity 1000, rare;
- source и flowing fluid;
- блок `thaumcraft:purifying_fluid`;
- ведро `thaumcraft:tc4_bucket_pure`;
- dispenser behavior для ведра;
- translucent client render layer;
- исходная анимированная `fluidpure.png` для still и flowing;
- включение в water fluid tag как современный аналог водного material-контракта TC4.

При столкновении с source-блоком игрок без Warp Ward получает эффект по исходной формуле:

`duration = min(32000, 200000 / max(1, floor(sqrt(permanentWarp))))`.

После этого source удаляется. Неоригинальный activation sound удалён; сохранены только белые пузырьки и редкий `lavapop` из TC4.

## 2. Bath Salts

`tc4_bath_salts` больше не является плоским предметом-зеркалом:

- зарегистрирован собственный `BathSaltsItem`;
- срок жизни выброшенного предмета — 200 тиков;
- по `ItemExpireEvent` соль заменяет только source обычной vanilla water на Purifying Fluid;
- flowing water, waterlogged-состояния и уже очищающая жидкость не подходят.

Crucible-рецепт `BathSalts` сохранён с точным research/catalyst/aspect-контрактом оригинала и продолжает отображаться в Alchemy JEI.

## 3. Arcane Spa

Добавлены `ArcaneSpaBlock` и `ArcaneSpaBlockEntity` под сохранённым ID `tc4_block_arcane_spa`:

- tank 5000 mB;
- один слот, принимающий только Bath Salts;
- режим mix включён по умолчанию и сохраняется в NBT;
- проверка каждые 40 game ticks;
- redstone signal полностью останавливает выдачу;
- mix-mode требует 1000 mB vanilla water и одну соль, затем размещает Purifying Fluid;
- dispense-mode размещает хранимую world-placeable жидкость;
- если source уже стоит над Spa, жидкость расширяется по поддерживаемой зоне 5×5;
- в ultra-warm измерении vanilla water не размещается;
- item/fluid automation открыта со всех граней, кроме верхней;
- tank, salt slot и mode сохраняются и отправляются клиенту через update packet;
- при разрушении соль выпадает.

Пока вместо исходного GUI используется временное прямое управление: Shift + ПКМ переключает режим, соль вставляется предметом, fluid containers обслуживаются через `FluidUtil`, пустая рука извлекает соль.

## 4. Исправлен фактически сломанный ArcaneSpa recipe

Во время сверки обнаружена отдельная ошибка старого data-файла. В `tc4_arcanespa.json` были перепутаны символы и отсутствовали piston/jar, а catalyst был obsidian — предметом, которого нет в шаблоне. При текущей логике Arcane Workbench такой рецепт не мог нормально совпасть с сеткой.

Теперь строка `ConfigRecipes.java:879` перенесена точно:

- pattern `QIQ / SJS / SPS`;
- `P = piston`;
- `J = essentia jar`;
- `S = arcane stone`;
- `Q = quartz block`;
- `I = iron bars`;
- cost `AQUA 16 / ORDO 8 / TERRA 4`;
- research `ARCANESPA`;
- результат — функциональный `tc4_block_arcane_spa`.

JEI использует явный symbol map и должен показывать все девять позиций сетки без прежнего сдвига. Это утверждение пока статическое: клиент JEI не запускался.

## 5. Связь с Warp и Sanity Soap

В 11.62.67 `SanitySoapItem` уже содержал hook для игрока, стоящего в `purifying_fluid`. Теперь этот ID принадлежит реальному fluid block, поэтому цепочка получает исполняемый источник для дополнительного шанса удаления sticky Warp.

## 8. Ограничения

Gradle Wrapper снова завершился до `compileJava`: `UnknownHostException: services.gradle.org` при попытке скачать Gradle 7.5.1. Поэтому:

- Java API-совместимость с Forge 43.5.2 не подтверждена компилятором;
- JAR не создан;
- gameplay, visual, multiplayer и save/reload тесты не выполнены;
- исходный Arcane Spa Menu/Screen и GUI-текстура ещё не подключены.

Рекомендуемый следующий блок: перенести `ContainerSpa`/`GuiSpa`, затем выполнить runtime-протокол Purifying Fluid/Spa; после этого продолжить essentia transport и алхимические устройства.
