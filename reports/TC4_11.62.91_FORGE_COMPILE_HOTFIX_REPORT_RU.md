## 1. Результат нового compiler pass

GitHub Actions прошёл все ранние проверки и дошёл до настоящей задачи `:compileJava`.
Компилятор показал одну ошибку и 100 предупреждений; сборка завершилась за 43 секунды.

Ошибка находилась в `TC4WispEntity.java:248`:

```text
incompatible types: EntityType<TC4WispEntity>
cannot be converted to EntityType<? extends Monster>
```

Падающий вызов:

```java
Monster.checkMonsterSpawnRules(type, level, reason, pos, random)
```

## 2. Корневая причина

`TC4WispEntity` наследуется от `FlyingMob` и реализует `Enemy`, но не является
наследником `Monster`. Статический helper `Monster.checkMonsterSpawnRules`
принимает только `EntityType<? extends Monster>`. Поэтому полный набор аргументов
был правильным, но generic type bound оставался несовместимым.

Менять Wisp на `Monster` нельзя без необоснованной замены иерархии и поведения
летающей сущности.

## 3. Исправление 11.62.91

Вместо несовместимого helper вызова сохранён тот же hostile-spawn контракт через
доступные независимые проверки:

```java
if (level.getDifficulty() == Difficulty.PEACEFUL
        || !Monster.isDarkEnoughToSpawn(level, pos, random)
        || !Mob.checkMobSpawnRules(type, level, reason, pos, random)) {
    return false;
}
```

Сохранены без изменений:

- наследование Wisp от `FlyingMob`;
- запрет естественного hostile spawn в Peaceful;
- vanilla darkness rule;
- общий Mob placement/obstruction rule;
- радиус подсчёта соседних Wisp 16 блоков;
- ограничение `nearby < 8`;
- biome modifier, вес и размер группы;
- все контракты Wisp parity 11.62.87.

## 4. Защита от повторения

Добавлен `tools/tc4_116291_wisp_flying_mob_spawn_rules_compile_guard.py`:

- фиксирует наследование `FlyingMob`;
- запрещает передачу Wisp EntityType в `Monster.checkMonsterSpawnRules`;
- требует Peaceful gate;
- требует `Monster.isDarkEnoughToSpawn`;
- требует `Mob.checkMobSpawnRules`;
- сохраняет radius/cap Wisp;
- запускается до Gradle в build/release workflows.

Общий `forge_1192_compile_api_guard.py` теперь также ловит этот type-bound regression.
Сохранённый guard 11.62.90 обновлён так, чтобы продолжать запрещать старый
трёхаргументный overload, но не требовать новый несовместимый вызов.
