# TC4 v11.63.54 — P1 runtime-контракты блок-сущностей

## Цель этапа

После P0-контрактов v11.63.53 следующий доказуемый пробел находился в четырёх уже реализованных системах блок-сущностей. Их исходный код существовал, но ключевая серверная семантика не входила в обязательный набор Forge GameTest.

Этап v11.63.54 добавляет отдельные server-only контракты для Tallow Candle, Hungry Chest, Brain in a Jar и обычных Magic Mirrors. Он не объявляет визуальный, сетевой или multiplayer-паритет без реального JAR и запуска тестового сервера.

## 1. Tallow Candle

Добавлен тест `tallowCandleKeepsSupportShapeAndInfusionStabilizerContract`.

Он фиксирует:

- использование специального `TallowCandleBlock`;
- реализацию маркера `InfusionStabilizer`;
- допустимость установки на твёрдую центральную опору;
- отсутствие collision shape;
- создание `TallowCandleBlockEntity`;
- недопустимость состояния после удаления опоры.

Этот контракт закрывает серверную форму и роль стабилизатора, но не проверяет частицы, цвет, модель и фактический вклад в долгую infusion-сессию.

## 2. Hungry Chest

Добавлен тест `hungryChestEatsPersistsAndExposesSingleInventory`.

Он проверяет:

- размещение канонического Hungry Chest;
- поглощение dropped `ItemEntity`;
- сохранение count, custom name и произвольного NBT;
- eat/lid event;
- Forge item-handler capability;
- сохранение содержимого через NBT round-trip.

Тест использует один именованный стек из пяти алмазов с контрольным NBT `TC4Proof=116354`.

## 3. Brain in a Jar

Добавлен тест `brainJarAbsorbsPersistsAndReportsComparatorExperience`.

Он закрепляет:

- поглощение соприкасающейся сферы опыта;
- точное накопление 37 XP;
- исходную шкалу comparator output;
- сохранение накопленного опыта через NBT round-trip.

Клиентская анимация мозга, звук и визуальная отдача опыта остаются отдельным runtime/client этапом.

## 4. Magic Mirrors

Добавлен тест `magicMirrorLinksTransportsAndPersistsQueuedStacks`.

Он проверяет:

- создание взаимной пары зеркал;
- транспорт tagged item entity;
- удаление исходной сущности;
- очередь на целевой стороне;
- count-based instability источника;
- сохранение очереди и количества предметов через NBT;
- двустороннее очищение ссылки при invalidation.

Этот тест выполняется в одном измерении. Cross-dimension, chunk unload, automation races и multiplayer остаются незакрытыми.

## Сборка и runtime-статус

Подтверждённый JAR v11.63.54 не создан. Wrapper остановился до конфигурации проекта:

```text
Downloading https://services.gradle.org/distributions/gradle-7.5.1-bin.zip
java.net.UnknownHostException: services.gradle.org
GRADLE_RC=1
```

В окружении установлен OpenJDK 21.0.10. Целевой Forge 1.19.2 проект должен быть повторно собран на Java 17. Поэтому все 28 тестов текущей версии имеют честный статус **NOT_TESTED**.

Подтверждённый Java 17 baseline v11.63.51 с 18/18 PASS не переносится автоматически на изменённые v11.63.52–54.
