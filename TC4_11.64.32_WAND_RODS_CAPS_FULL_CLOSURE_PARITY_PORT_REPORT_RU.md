# TC4 v11.64.32 — жезлы, стержни и наконечники: полное source/resource-закрытие

## Граница раунда

Единственная закрываемая механика — базовая система casting wands: `WandItem`, rod/staff rod, wand cap, ёмкость vis, cap discounts, sceptre-множители, элементальное и primal-восстановление, component NBT, сборочные варианты, creative/JEI-каталог, название, свойства предмета и базовый rod/cap-rendering. Фокусы, Focus Pouch, их эффекты, upgrades и отдельная first-person focus-анимация в этот раунд не закрывались.

## Матрица расхождений и исправлений

1. **Creative-каталог был раздут до 216 комбинаций.** Оригинальный `ItemWandCasting#getSubItems` показывает ровно четыре демонстрационных предмета: wood/iron, greatwood/gold, silverwood/thaumium и silverwood/thaumium sceptre. Production-каталог возвращён к этим четырём вариантам.
2. **Жезлы наследовали обычный stack size и common rarity.** Все три casting-wand registry entry теперь имеют `stacksTo(1)` и `Rarity.UNCOMMON`, как исходный `ItemWandCasting`.
3. **Component NBT дублировался во вложенном compound `Wand`.** Каноническая запись теперь содержит root keys `rod` и `cap`. Старый nested-формат читается только для одноразовой миграции и удаляется после нормализации.
4. **Отсутствовал исходный staff weapon modifier.** Staff rod теперь добавляет ровно `+6.0` attack damage только в main hand с исходным стандартным UUID modifier.
5. **Название было собрано жёстко заданным английским текстом.** Оно теперь формируется через `Component.translatable` из rod/cap/base wand localization keys и общего шаблона `item.thaumcraft.wand_name_format`.
6. **Canonical rod/cap items были пассивными generic items.** Greatwood/Silverwood rods и Iron/Gold/Thaumium caps теперь создаются функциональным `TC4WandComponentItem` и несут subtype identity для recipes/JEI.
7. **Пять migration-alias ID дублировали компоненты в creative/search.** Registry ID сохранены для старых миров, но скрыты из каталога. `wooden_wand_core` также остаётся только migration ID: оригинальный Wood Rod использует обычную палку.
8. **На компонентах и жезле были придуманные debug/tooltips.** Удалены нестандартные family/tag/regen-подсказки; оставлены только актуальные vis/capacity/focus сведения production-предмета.
9. **Формулы были размазаны по Forge-классам.** Capacity, sceptre 150%, cap modifier, sceptre discount `-0.1` с floor `0.1`, порог регенерации и перевод vis→centivis вынесены в dependency-free `TC4WandComponentMath` и подключены к настоящему production-пути.
10. **Название regen-константы было неверным.** `REGEN_AMOUNT_CENTIVIS = 1` переименовано в `REGEN_AMOUNT_VIS = 1`. Это не новый gameplay-фикс: существующий `WandItem#addVis` уже умножал значение на 100, поэтому фактически восстанавливался правильный 1 vis. Раунд добавляет ясный контракт и тест против будущей регрессии.
11. **Полный rod/cap-каталог не имел отдельного строгого closure-контракта.** Добавлен контракт для 18 rod/staff-типов, 6 cap-типов, 17 rod component variants, 9 cap component variants, четырёх creative-жезлов и исходных параметров каждого элемента.

## Gradle и runtime

Команда `./gradlew build --no-daemon --console=plain` действительно запускалась. Gradle Wrapper начал загрузку Gradle 7.5.1, но завершился с:

- `java.net.UnknownHostException: services.gradle.org`;
- exit code `1`;
- активная Java среды — OpenJDK 21.0.10 вместо требуемой JDK 17.

Поэтому Forge compile, JAR, client, dedicated server, multiplayer, реальное потребление vis, tick regeneration, staff combat, creative search, JEI и runtime GameTests не подтверждены.
