# Thaumcraft Legacy Rebuild 1.19.2 — v11.63.19

## Стадия

**Thaumostatic Harness / Таумостатический ранец — визуальная модель на игроке.**

Целевая платформа: Minecraft 1.19.2, Forge 43.5.2, Java 17. Эталон поведения и геометрии: Thaumcraft 4.2.3.5 для Minecraft 1.7.10. Стадия продолжает функциональный перенос ранца и пояса из v11.63.18 и закрывает зафиксированный там пробел worn-model renderer.

## Реализовано

### Нагрудная оболочка

- `HoverHarnessItem.initializeClient` подключает отдельный `IClientItemExtensions`.
- `TC4HoverHarnessArmorModel` создаёт только исходную оболочку туловища: куб 8×12×4, UV-смещение 16/16 и раздувание 0.6 пикселя.
- Атлас оболочки сохранён как `textures/models/hoverharness.png`, 128×64.
- Голова, шляпа, руки и ноги модели скрыты, чтобы стандартный chestplate не добавлял чужую геометрию.
- Модель регистрируется через `EntityRenderersEvent.RegisterLayerDefinitions` и запекается один раз в `EntityRenderersEvent.AddLayers`.

### Исходная OBJ-модель

- Все 124 треугольника `textures/models/hoverharness.obj` преобразованы в статический массив вершин.
- Для каждой вершины сохранены координаты, UV и нормаль; V-координата перевёрнута тем же способом, что и в уже перенесённых OBJ-моделях проекта.
- Треугольники выдаются как независимые вырожденные quad-примитивы для совместимости с entity render type.
- Воспроизведён порядок исходных преобразований: масштаб 0.1, поворот −90° вокруг X, перенос `(0, 0.33, −3.7)`.
- Используется исходная текстура `hoverharness2.png`, 64×64.
- Дополнительный слой устанавливается на все зарегистрированные варианты скина игрока, включая `default` и `slim`.

### Активные эффекты

- Эффекты показываются только при `HoverHarnessItem.isHoverEnabled(stack)`.
- Восстановлены две окружности: белая размером 2.5 и развёрнутая фиолетовая размером 1.5 с разносом 0.03 по глубине.
- Прямо выбираются 16 вертикальных кадров исходной текстуры `lightningring.png`; освещение принудительно full-bright.
- Добавлена локальная восьмисегментная молния с детерминированным seed по ID игрока и трёхтиковому интервалу. Она не хранит глобальных клиентских таймеров и исчезает сразу после выключения режима.

## Современные адаптации

- Устаревший `AdvancedModelLoader` заменён проверяемым статическим vertex-массивом.
- `GL11`-состояния и `FXLightningBolt` не имеют прямого безопасного аналога в современном player layer. Окружности используют `RenderType.eyes`, а молнии — `RenderType.lines`.
- Оригинальная молния выбирала случайный блок raycast в радиусе 6 и обновлялась каждые 50–100 мс. Современная версия использует локальный детерминированный endpoint и обновление каждые три игровых тика; это сознательная визуальная адаптация, а не утверждение byte-for-byte runtime parity.
- Поворот при приседании не дублируется вручную: слой наследует текущий transform `PlayerModel.body`, что также позволяет следовать плаванию и другим современным позам.

## Компиляция и runtime

`./gradlew compileJava --stacktrace --no-daemon` снова остановился до запуска Java-компилятора. Gradle Wrapper не смог разрешить `services.gradle.org` при загрузке Gradle 7.5.1 (`UnknownHostException`). В среде установлен OpenJDK 21.0.10, отдельный JDK 17 не найден.

JAR не создан. До успешной компиляции и запуска клиента нельзя считать подтверждёнными масштаб на игроке, culling/blending, default/slim, crouch/swim/Elytra, resource reload, shaders, dedicated server и отображение двух игроков.
