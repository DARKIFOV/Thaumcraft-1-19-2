# TC4 v11.63.56 — runtime-контракты транспорта essentia по трубам

## Цель этапа

Продолжить портирование после v11.63.55 и закрыть следующий доказуемый P1-пакет server-side механик: обычные, фильтрующие, ограничивающие, направленные и буферные essentia-трубы. Этап не объявляет runtime PASS без Java 17 build и реального Forge GameTest server.

## Исправленный runtime-дефект

В `EssentiaTubeBlockEntity.restoreBufferForNetwork` rollback раньше возвращал аспект в `bufferAspects`, но:

- не ограничивал восстановление исходной ёмкостью буфера 8;
- не обновлял `essentiaType`;
- не обновлял `essentiaAmount`;
- поэтому API смешанного буфера и legacy transport-view могли расходиться сразу после rollback.

Теперь восстановление:

1. допускается только для subtype `BUFFER`;
2. ограничивается свободным остатком `BUFFER_CAPACITY = 8`;
3. обновляет authoritative `AspectList`;
4. сразу синхронизирует `essentiaType` и `essentiaAmount`;
5. вызывает block-entity sync.

## Добавленные GameTests

### Normal и Restricted Tube

Проверяется propagation suction от обычной банки:

- normal tube: `32 - 1 = 31`;
- restricted tube: половина соседнего suction, `32 / 2 = 16`.

### Filtered Tube

Проверяется:

- тип suction совпадает с label/filter;
- suction от filtered jar равен `64 - 1 = 63`;
- разрешён только выбранный аспект;
- после смены filter на несовместимый аспект прежний suction сбрасывается.

### One-way Tube

Проверяется:

- suction проходит только через разрешённую оригинальной ориентацией сторону;
- после смены facing запрещённое направление не сохраняет старый suction.

### Essentia Buffer

Проверяется:

- смешанное хранение аспектов;
- точная ёмкость 8;
- excess rollback не переполняет буфер;
- live transport amount совпадает с mixed storage;
- suction остаётся untyped;
- choke state `2` отключает suction стороны;
- содержимое и choke переживают NBT round-trip;
- drain и rollback синхронно меняют transport-view.

## Граница доказательства

Этот этап не закрывает:

- произвольные длинные сети;
- равные/конфликтующие suction-пути;
- venting и смену аспектов под нагрузкой;
- valves, bellows и автоматическое choke-routing;
- все 48 аспектов в одной матрице;
- furnace/alembic/centrifuge/Thaumatorium end-to-end;
- chunk unload/reload;
- simultaneous transfer races;
- multiplayer/network и client FX.
