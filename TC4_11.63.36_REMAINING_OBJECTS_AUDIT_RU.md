# Аудит оставшихся TC4-ID — v11.63.36

Этот аудит отвечает на вопрос «сколько осталось» по воспроизводимому статическому критерию: ID всё ещё создаётся общим `TC4ResearchComponentItem`, а не отдельным классом или заранее зарегистрированным функциональным объектом.

| Показатель | Значение |
|---|---:|
| Записей в de-metadata mirror | 294 |
| ID с отдельной веткой `createItem` | 224 |
| Заранее зарегистрированные функциональные ID | 16 |
| Намеренно пропущенные дубли BlockItem | 4 |
| **Оставшиеся generic fallback ID** | **50** |
| Из них предметоподобные ID | 8 |
| Из них блочные migration-alias ID | 42 |

## Как читать число

**50 — это верхняя техническая оценка, а не число пятидесяти самостоятельных механик.** После v11.63.36 простые ресурсные и экипировочные семейства из предметного fallback уже удалены. Текущий остаток состоит из восьми уникальных предметных систем и 42 старых блочных migration-alias; часть блочных alias уже указывает на перенесённые блоки и требует прежде всего нормализации регистрации/миграции, а не повторной реализации блока.
