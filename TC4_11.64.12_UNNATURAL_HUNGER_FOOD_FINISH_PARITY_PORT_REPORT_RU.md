# Отчёт v11.64.12: Unnatural Hunger food-finish parity

**Дата:** 22.07.2026  
**Проект:** Thaumcraft Legacy Rebuild, Minecraft Forge 1.19.2  
**Эталон:** Thaumcraft 4.2.3.5 для Minecraft 1.7.10  
**Статус:** source/static contract исправлен; Gradle build и Forge runtime **NOT VERIFIED**.

## 1. Найденное расхождение

В порте эффект `Unnatural Hunger` уже увеличивал истощение игрока и содержал список curative items, но отсутствовал отдельный обработчик завершения еды из оригинального `EventHandlerEntity.finishedUsingItem`.

Из-за этого не воспроизводились четыре наблюдаемых правила TC4:

1. Гнилая плоть или мозг зомби должны уменьшать оставшуюся длительность эффекта на **600 тиков**.
2. Они одновременно уменьшают amplifier ровно на **1**.
3. Эффект возвращается только когда после уменьшения `duration > 0` и `amplifier >= 0`; поэтому level 0 снимается полностью одной особой едой.
4. Обычная еда не отменяется, но выводит `warp.text.hunger.1`; особая еда выводит `warp.text.hunger.2`.

Один лишь `MobEffectInstance#setCurativeItems` этот оригинальный event-path не заменяет.

## 2. Доказательство по оригиналу

Использован локальный oracle:

- `reference/Thaumcraft4-1.7.10-4.2.3.5-source.zip`;
- `thaumcraft/common/lib/events/EventHandlerEntity.java`, метод `finishedUsingItem(PlayerUseItemEvent.Finish event)`;
- `thaumcraft/common/lib/potions/PotionUnnaturalHunger.java`.

Оригинальный обработчик выполняет:

- сравнение съеденного предмета с rotten flesh и `ConfigItems.itemZombieBrain`;
- `amp = currentAmplifier - 1`;
- `duration = currentDuration - 600`;
- удаление текущего эффекта;
- повторное добавление только при `(duration > 0) && (amp >= 0)`;
- очистку curatives повторно добавленного эффекта и добавление только rotten flesh;
- зелёное курсивное сообщение `warp.text.hunger.2` для особой еды;
- красное курсивное `warp.text.hunger.1` для остальных `ItemFood` без отмены еды.

Для Forge 1.19.2 выбран `LivingEntityUseItemEvent.Finish`: его `getItem()` предоставляет копию предмета до использования, тогда как результат уже отражает завершённое использование. Это соответствует точке вмешательства оригинального `PlayerUseItemEvent.Finish`.

## 3. Изменения production-кода

### `WarpEvents.java`

Добавлен server-side обработчик `onItemUseFinish(LivingEntityUseItemEvent.Finish)`:

- работает только для `ServerPlayer` с активным `UNNATURAL_HUNGER`;
- распознаёт `minecraft:rotten_flesh` и `thaumcraft:tc4_brain`;
- делегирует арифметику в `TC4WarpRuntimeParity.unnaturalHungerAfterCurative`;
- сначала удаляет старый эффект;
- при допустимом результате создаёт новый ambient instance с уменьшенными значениями;
- после повторного добавления сохраняет только rotten flesh в explicit curative list, как оригинал;
- выводит точные translation keys и цвета;
- для обычной съедобной еды показывает предупреждение, не меняя result stack и не отменяя питание.

### `TC4WarpRuntimeParity.java`

Добавлены source-linked значения:

- `UNNATURAL_HUNGER_CURATIVE_DURATION_REDUCTION_TICKS = 600`;
- `UNNATURAL_HUNGER_CURATIVE_AMPLIFIER_REDUCTION = 1`;
- `unnaturalHungerAfterCurative(...)`;
- record `UnnaturalHungerReduction`.

Contract version поднят до `11.64.12`.
