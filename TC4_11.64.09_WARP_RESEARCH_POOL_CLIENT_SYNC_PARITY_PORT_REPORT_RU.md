# Отчёт о раунде портирования v11.64.09 — синхронизация пула аспектов после warp grantResearch

Minecraft 1.19.2 · Forge 43.5.2 · целевая Java 17 · эталон Thaumcraft 4.2.3.5 (1.7.10)

## Найденное расхождение

В оригинальном `thaumcraft/common/lib/WarpEvents.java` метод `grantResearch` после каждого случайного начисления примарного аспекта:

1. добавляет игроку `+1` в aspect pool;
2. сразу отправляет `PacketAspectPool` этому игроку;
3. планирует сохранение research-данных.

В порте v11.64.08 оба production-пути — обычные warp-события (`WarpEvents`) и вехи `ELDRITCHMINOR`/`ELDRITCHMAJOR` (`TC4EldritchProgression`) — изменяли серверный persistent NBT через `PlayerAspectKnowledge.addPool`, но не вызывали `ThaumcraftNetwork.syncAspectKnowledge`. Отправлялся только research/warp sync, который не содержит aspect pool. В результате клиентский Research Table/Thaumonomicon мог видеть старый баланс до более поздней полной синхронизации.

## Исправление production-кода

Добавлен общий production-класс `warp/TC4WarpResearchGrant.java`:

- точная формула оригинала: `1 + random.nextInt(times)`;
- точный порядок шести примарных аспектов: Aer, Terra, Ignis, Aqua, Ordo, Perditio;
- каждое выбранное значение добавляет `+1` через `PlayerAspectKnowledge.addPool`;
- после завершения пакета вызывается `ThaumcraftNetwork.syncAspectKnowledge(player)`.

Оригинал отправлял один `PacketAspectPool` на каждую единицу. Порт 1.19.2 отправляет один агрегированный `AspectKnowledgeSyncPacket` после всей серии. Итоговый клиентский пул совпадает, при этом исключается серия лишних пакетов.

Оба старых приватных метода `grantResearch` теперь делегируют в общий production-путь:

- `event/WarpEvents.java`;
- `eldritch/TC4EldritchProgression.java`.
