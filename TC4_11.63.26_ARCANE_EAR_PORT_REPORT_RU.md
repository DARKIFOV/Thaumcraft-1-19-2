# Thaumcraft Legacy Rebuild 1.19.2 — v11.63.26

## Стадия

**Arcane Ear / Магическое ухо** — перенос `TileSensor` и варианта metadata 1 блока `BlockWoodenDevice` из Thaumcraft 4.2.3.5 на Forge 1.19.2.

Базовая линия: загруженный пользователем исходник v11.63.25 с уже перенесённым Arcane Bore. Загруженный JAR v11.63.25 использовался только как проверяемый входной артефакт; новый JAR v11.63.26 не заявляется без фактического полного Gradle build.

## Реализовано

- `tc4_block_arcane_ear` исключён из generic fallback и зарегистрирован как настоящий `ArcaneEarBlock` с `ArcaneEarBlockEntity`.
- Сохраняются выбранная нота `0..24`, тембр `0..4` и оставшееся время импульса.
- Правый клик циклически повышает ноту и воспроизводит текущую настройку.
- Материал блока снизу выбирает исходные пять инструментов: обычный/harp, stone/bass drum, sand/snare, glass/hat, wood/bass.
- Подписка на Forge `NoteBlockEvent.Play` передаёт реально сыгранные ноты всем загруженным Магическим ушам измерения.
- Совпадение требует одинаковых ноты и инструмента и расстояния не более 64 блоков (`distance² <= 4096`).
- Совпавшая нота выдаёт десятиковый redstone-сигнал силы 15; повторная нота обновляет таймер.
- При окончании импульса уведомляются соседи самого блока и блока под ним.
- Блок над Ухом подавляет звук и частицу, но не удалённое обнаружение и redstone-импульс, как в TC4.
- Ручная проверка создаёт звук громкостью 3 с высотой `2^((note-12)/12)` и note-частицу; удалённое срабатывание создаёт только частицу.
- Событие без звука использует современный безопасный event ID 5 вместо старого `-1`, поскольку block-event ID передаётся по сети как байт.
- Восстановлена десятичастная геометрия: основание, центральная стойка и восемь деталей четырёх раструбов.
- POWERED-state переключает исходные `arcaneearside/top off/on`; bell-текстуры остаются неизменными.
- Добавлены blockstate, три block model, настоящий 3D item model, loot table, axe tag и нормальные EN/RU block/item names.

## Компиляция и runtime

Локальная среда текущей сессии предоставляет OpenJDK 21, а Gradle Wrapper должен скачать Gradle 7.5.1 с `services.gradle.org`. Доступ к этому хосту в контейнере отсутствует. Поэтому новый `compileJava`/`build` не объявляется успешным, JAR v11.63.26 не создан, а все шесть новых runtime-сценариев остаются `NOT_TESTED`.

Для окончательной проверки на Windows требуется JDK 17 и:

```powershell
.\gradlew.bat compileJava --no-daemon --console=plain
.\gradlew.bat clean build --no-daemon --console=plain
```

Успех допустимо фиксировать только при `BUILD SUCCESSFUL`, коде 0 и новом проверенном JAR в `build\libs`.
