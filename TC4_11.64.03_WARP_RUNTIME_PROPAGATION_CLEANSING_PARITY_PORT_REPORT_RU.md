# Thaumcraft Legacy Rebuild v11.64.03 — Warp Runtime Propagation and Cleansing Parity

## Цель раунда

Исправить два подтверждённых расхождения в серверной warp-механике после того, как v11.64.02 восстановила правильное распределение research warp по permanent/sticky buckets:

1. лечение вторично распространённого `Infectious Vis Exhaust`;
2. расход Sanity Soap в творческом режиме.

Дополнительно исходные интервалы, шансы и формула Warp Ward вынесены в единый production-контракт.

## Поведение оригинального Thaumcraft 4.2.3.5

### Infectious Vis Exhaust

`PotionInfectiousVisExhaust` распространяется каждые 40 тиков в радиусе 4 блоков:

- amplifier выше нуля создаёт новый infectious-эффект с `amplifier - 1`;
- amplifier ноль создаёт обычный `Vis Exhaust` amplifier 0;
- длительность распространённого эффекта — 6000 тиков;
- распространённые `PotionEffect` создавались напрямую и сохраняли стандартное лечение молоком;
- только исходный эффект, созданный таблицей `WarpEvents`, очищал список curative items.

### Sanity Soap

`ItemSanitySoap`:

- используется 200 тиков и завершается после более чем 195 использованных тиков;
- всегда уменьшает размер ItemStack на единицу, включая creative;
- имеет базовый шанс снять один sticky warp `0.33`;
- Warp Ward добавляет `0.25`;
- нахождение в очищающей жидкости добавляет ещё `0.25`;
- весь temporary warp удаляется при успешном завершении использования.

### Purifying Fluid

Длительность Warp Ward зависит только от permanent warp:

```text
min(32000, 200000 / max(1, floor(sqrt(permanentWarp))))
```

## Ошибки предыдущего порта

- `spreadVisExhaustion` вызывал `configureCuratives` и для вторичных эффектов. Из-за этого заражённые соседние сущности получали неизлечимый эффект, хотя в оригинале могли снять его молоком.
- `SanitySoapItem` не уменьшал ItemStack при `instabuild`, хотя TC4 выполнял прямой decrement независимо от creative.
- Числа `2000`, `40`, `4`, `6000`, `200`, `195`, `0.33`, `0.25`, `32000` и `200000` были распределены по production-классам без единого проверяемого источника.

## Реализация v11.64.03

Добавлен чистый source-linked контракт:

```text
com.darkifov.thaumcraft.warp.TC4WarpRuntimeParity
```

Production-код теперь использует его в:

- `WarpEvents` — интервал warp-проверки 2000 тиков;
- `TC4WarpMobEffect` — радиус, cadence, длительность и downgrade распространения;
- `SanitySoapItem` — use duration, completion threshold, sticky chance и расход stack;
- `PurifyingFluidBlock` — формула длительности Warp Ward.

Распространённый эффект создаётся обычным `new MobEffectInstance(...)` без очистки curatives. Исходный warp-event эффект по-прежнему проходит через `configureCuratives` и остаётся неизлечимым.
