# Аудит оставшихся TC4-ID — v11.63.38

Этот аудит отвечает на вопрос «сколько осталось» по воспроизводимому статическому критерию: ID всё ещё создаётся общим `TC4ResearchComponentItem`, а не отдельным классом или заранее зарегистрированным функциональным объектом.

| Показатель | Значение |
|---|---:|
| Записей в de-metadata mirror | 294 |
| ID с отдельной веткой `createItem` | 269 |
| Заранее зарегистрированные функциональные ID | 17 |
| Намеренно пропущенные дубли BlockItem | 4 |
| **Оставшиеся generic fallback ID** | **4** |
| Из них предметоподобные ID | 0 |
| Из них блочные migration-alias ID | 4 |

## Как читать число

**4 — это верхняя техническая оценка блочной миграции, а не число самостоятельных неперенесённых механик.** После v11.63.38 предметоподобных ID в общем fallback не осталось, а 38 из 42 блочных alias нормализованы. Текущий список состоит только из четырёх самостоятельных систем: лампы роста/плодородия и обычный/фокусированный пьедесталы жезла.
