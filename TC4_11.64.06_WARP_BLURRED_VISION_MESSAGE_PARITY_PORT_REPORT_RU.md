# Thaumcraft Legacy Rebuild v11.64.06 — Warp blurred-vision headache message parity

## Цель раунда

Восстановить оригинальное сообщение warp-события "blurred vision" (эффект замешательства) из TC4 4.2.3.5: строку `warp.text.8` («Surely there must be a way to stop these headaches?»).

## Подтверждённое расхождение

В оригинальном TC4 каждая из 15 строк `warp.text.1`..`warp.text.15` показывается конкретной веткой warp-события. Ветка эффекта замешательства (blurred vision) выводит `warp.text.8` про «головные боли».

В порте v11.64.05 ветка `effectRoll <= 36` накладывала эффект `BLURRED_VISION`, но НЕ вызывала `message(...)`. В результате `warp.text.8` оказалась единственной оригинальной warp-строкой, которую порт никогда не показывал, хотя сама строка уже присутствовала в перенесённых языковых файлах (`en_us.json` и др.).

Доказательства (в архиве, без обращения к сети):
- `src/main/resources/assets/thaumcraft/original_tc4_1710/lang/en_US.lang`: `warp.text.8` — строка про «headaches» (оригинальный оракул 1.7.10).
- `src/main/resources/assets/thaumcraft/lang/en_us.json`: та же строка уже перенесена.
- Статический анализ `WarpEvents.java`: из 15 строк использовались 14; `warp.text.8` не встречалась в Java ни разу.

## Production-исправление

1. `TC4WarpRuntimeParity`: добавлена source-linked константа `BLURRED_VISION_MESSAGE_KEY = "warp.text.8"`; `CONTRACT_VERSION = 11.64.06`.
2. `WarpEvents.applyOriginalEvent`: ветка blurred vision (`effectRoll <= 36`) теперь вызывает `message(player, TC4WarpRuntimeParity.BLURRED_VISION_MESSAGE_KEY)`, показывая оригинальную строку.
3. Добавлен production-аксессор `WarpEvents.usesWarpMessage(String)` c множеством `DISPLAYED_WARP_MESSAGES`, перечисляющим все реально выводимые строки (1..15), что позволяет guard/тесту доказать, что ни одна оригинальная warp-строка больше не теряется.

Поведение до/после:
- Было: игрок с эффектом blurred vision получал только визуальный эффект без чат-строки; `warp.text.8` не показывалась никогда.
- Стало: та же ветка выводит оригинальную строку `warp.text.8`, как в TC4 4.2.3.5.

## Новый GameTest-контракт

Добавлен обязательный метод `warpBlurredVisionUsesOriginalHeadacheMessage`:
- проверяет, что `BLURRED_VISION_MESSAGE_KEY == "warp.text.8"`;
- что ключ входит в оригинальную таблицу warp-строк;
- что production-таблица `WarpEvents.usesWarpMessage(...)` реально выводит эту строку.

Это поведенческий контракт (значение и связь с production-таблицей), а не подсчёт файлов. Метод существует в исходниках, но не считается runtime PASS без запуска Forge GameTest server на Java 17.

## Guard

`tools/tc4_116406_warp_message_parity_guard.py` проверяет:
- версии `build.gradle`/`mods.toml`/`CONTRACT_VERSION` >= 11.64.06;
- наличие константы и её вызов в ветке blurred vision;
- что все 15 строк присутствуют в множестве выводимых сообщений;
- совпадение текста `warp.text.8` в `en_us.json` и оракуле `original_tc4_1710`;
- наличие GameTest-метода и manifest-сценария; уникальность ID;
- `runtime_status = NOT_VERIFIED` в source evidence.

Старые guards v11.64.03/04/05 сделаны forward-compatible (принимают >= своей версии и совокупные счётчики GameTest/сценариев).
