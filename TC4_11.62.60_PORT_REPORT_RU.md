# Thaumcraft Legacy Rebuild 11.62.60 — отчёт по продолжению порта рецептов и JEI

## База проверки

- Порт: Forge 1.19.2, исходная ревизия 11.62.59.
- Эталон: исходники Thaumcraft 4.2.3.5, `thaumcraft/common/config/ConfigRecipes.java`.
- Новая ревизия: **11.62.60**.

Цель этого прохода — не добавлять новые «похожие» рецепты, а переносить только те рецепты TC4, для которых уже существуют реальные runtime-предметы/блоки порта и однозначно восстановлены входы, выходы, аспекты и форма.

## Изменения runtime и JEI

### Теги катализаторов тигля

`AlchemyRecipe` теперь принимает катализатор в четырёх формах:

```json
"catalyst": "minecraft:iron_ingot"
```

```json
"catalyst": "#forge:ores/tin"
```

```json
"catalyst": { "item": "minecraft:iron_ingot" }
```

```json
"catalyst": { "tag": "forge:ores/tin" }
```

Runtime-проверка использует `ItemStack#is(TagKey)`, поэтому любой предмет из тега работает в тигле и Thaumatorium.

### JEI

- Слот катализатора Alchemy теперь получает `recipe.catalystIngredient()` и показывает все варианты тега.
- В категориях Arcane, Alchemy и Infusion отображается исходный ключ рецепта TC4.
- Рецепты посоховых компонентов объединяются по исходному `tc4_key`: авторитетный генерируемый рецепт заменяет старую JSON-копию, поэтому JEI не должен показывать два варианта с разной ценой или формой.

## Сборка

Запуск `./gradlew compileJava --no-daemon --stacktrace` выполнен, но Gradle Wrapper не смог скачать Gradle 7.5.1 из-за отсутствия DNS/сетевого доступа к `services.gradle.org`:

```text
java.net.UnknownHostException: services.gradle.org
```

Это ограничение среды, а не результат Java-компиляции. Поэтому готовый JAR в этот пакет не включён и runtime-совместимость JEI следует подтвердить сетевой CI-сборкой и запуском клиента.

## Осознанно не перенесено в этом проходе

- `NodeChargeRelay`: старый resolver сводит meta 2 и meta 14 `blockMetalDevice` к одному `vis_relay`; сначала нужен отдельный runtime-блок зарядного реле.
- `WardedGlass`: текущая таблица пытается подменить оригинальный блок объектом аддона `tce_warded_glass`; это не строгий порт.
- Рецепты Hungry Chest, Brain Jar, Mirror и других отсутствующих runtime-блоков: JSON-затычки не добавлялись до реализации самих объектов.

Таким образом, этот проход увеличивает покрытие только проверяемыми оригинальными рецептами и одновременно удаляет несколько уже существовавших ложных соответствий.
