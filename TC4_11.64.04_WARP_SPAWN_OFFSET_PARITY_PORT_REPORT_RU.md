# Thaumcraft Legacy Rebuild 11.64.04 — исправленная parity смещений warp-спавна

## Причина исправления

Первая сборка `v11.64.04` ошибочно утверждала, что оригинальный TC4 применял к модулю 7–24 только булев знак. Это неверно. В `thaumcraft/common/lib/WarpEvents.java`, строки комментариев декомпиляции 356–358 и 388–390, каждая ось вычисляется как:

```text
random(7, 24) * random(-1, 1)
```

Обе границы второго вызова включаются. Поэтому исходы знака: `-1`, `0`, `1`. Нулевая ось является частью оригинальной механики.

## Что исправлено

- `WarpEvents.randomTc4AxisOffset` снова получает включительный `signRoll` через `Mth.nextInt(random, -1, 1)`.
- `TC4WarpRuntimeParity.signedSpawnOffset(int magnitude, int signRoll)` сохраняет отрицательный, нулевой и положительный исходы и нормализует неожиданные значения только по знаку.
- Удалён неправильный GameTest `warpSpawnOffsetsKeepOriginalSignedMinimum`.
- Добавлен `warpSpawnOffsetsPreserveOriginalTriStateSign`.
- Manifest ID заменён на `gametest.warp_spawn_signed_tristate_offsets`.
- Source-evidence JSON, старый Warp/Eldritch guard, focused guard и pure-Java self-test исправлены.
- Документация, START_HERE, future-chat prompt и package manifest синхронизированы.
- Итоговый ZIP упакован плоско: `build.gradle` находится в корне, без лишней папки `port/`.
