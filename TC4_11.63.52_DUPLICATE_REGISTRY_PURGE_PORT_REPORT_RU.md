# TC4 v11.63.52 — удаление точных дублей registry и канонизация предметов

**Цель этапа:** убрать из регистрации предметы, которые доказуемо представляли тот же объект TC4 под вторым ID, не удаляя самостоятельные предметы и не ломая старые сохранения.

## Выполнено

- Удалены **63** точных item-alias ID.
- Удалены **63** дублирующих item-model JSON и соответствующие дублирующие ключи локализации.
- Все активные datapack-рецепты и runtime-ссылки переведены на канонические ID.
- `TC4ResearchItems.registerAll` больше не регистрирует удалённые aliases.
- Добавлен `MissingMappingsEvent` для remap старых registry-имён на уже зарегистрированные канонические предметы.
- Сохранён глубокий мигратор `ItemStack`: count, damage, обычный NBT, enchantments, custom name и сериализованные Forge capabilities переносятся через замену поля `id`; вложенные item-handler инвентари обходятся до глубины 4.
- Добавлен обязательный server-only GameTest `exactDuplicateLegacyItemAliasesAreNotRegistered`: проверяет отсутствие всех старых ID и наличие всех канонических целей.

## Что намеренно не удалено

Удаление выполнялось только при доказанном равенстве объекта. Сходные названия или текстуры сами по себе не считались достаточным основанием. Сохранены самостоятельные объекты, включая:

- `tc4_dust` (Salis Mundus) и отдельную алхимическую пыль;
- Gold Key и Awakened Crimson Key;
- Runed Tablet и Eldritch Relic;
- обычные и infused scribing tools;
- варианты культовой брони, если у них различается runtime-поведение, экипировка или progression-роль.

После очистки автоматический аудит английских item-названий находит только одну одинаковую подпись — `Scribing Tools` у обычной и infused-версии. Это два разных предмета, поэтому registry-слияние не выполнялось.
