# TC4 11.63.61 — Infusion Matrix: pause/resume и locked-recipe parity

## Закрытый пакет

Восстановлено исходное поведение матрицы при временной потере структуры алтаря и при отсутствии pedestal-компонента после старта рецепта.

### Потеря структуры

- нарушение центральной опоры или одной из четырёх диагональных колонн больше не переводит процесс в terminal failure;
- `active` сбрасывается, но locked recipe, catalyst, pending components, essentia, instability, owner и progress сохраняются;
- после ремонта структуры повторная активация жезлом возобновляет тот же craft;
- отдельный Warp за structural pause не начисляется.

### Locked recipe

- после успешного старта матрица больше не переоценивает рецепт по текущему набору предметов на pedestal;
- временно снятый или отсутствующий компонент оставляет процесс в ожидании;
- матрица продолжает искать именно сохранённые pending ingredients исходного рецепта.

## Реализация

Добавлены:

- `TC4InfusionPauseResumeParity`;
- `pauseForInvalidStructure()` без очистки crafting-state;
- `reactivatePausedInfusion()` для возобновления через жезл;
- четыре обязательных server-only GameTest-контракта;
- guard `tc4_116361_infusion_pause_resume_runtime_guard.py`.

Удалена ошибочная post-start проверка `lockedRecipeStillMatchesCurrentPedestals`, которая могла отменить уже выбранный рецепт до первого component pull.
