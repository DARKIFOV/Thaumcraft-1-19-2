# Thaumcraft Legacy Rebuild 11.63.04 — Brainy Zombie family

**Целевая платформа:** Minecraft 1.19.2, Forge 43.5.2, Java 17  
**Эталон:** Thaumcraft 4.2.3.5  
**Статус:** source/resource contracts реализованы; runtime и реальная Gradle-компиляция в текущей среде не подтверждены.

## Перенесённый блок

Добавлены две отсутствовавшие сущности оригинального TC4:

- `thaumcraft:brainy_zombie` — **Злой зомби / Angry Zombie**;
- `thaumcraft:giant_brainy_zombie` — **Яростный зомби / Furious Zombie**.

До этой стадии в проекте уже существовали их aspect mappings и legacy entity scan aliases, однако реальные `EntityType`, renderer, spawn eggs и поведение отсутствовали. Поэтому скрытое исследование `JARBRAIN` нельзя было разблокировать естественным сканированием этих существ.

## Brainy Zombie

Восстановлены подтверждённые оригинальным `EntityBrainyZombie` параметры:

- максимальное здоровье: `25`;
- базовый урон: `5`;
- добавочная броня: `3`, с современным attribute-представлением;
- шанс вызова подкрепления: `0`;
- обычный zombie XP без придуманного дополнительного значения;
- три независимые 50-процентные попытки дропа одной гнилой плоти;
- мозг по формуле `nextInt(10) - looting <= 4`;
- пустая стандартная loot table, чтобы vanilla-дроп зомби не добавлялся поверх оригинального `dropFewItems`.

Естественное появление добавлено через `forge:add_spawns`:

- biomes: `#minecraft:is_overworld`;
- weight: `10`;
- pack: `1–1`.

Это соответствует общей ветке `ConfigEntities`, добавлявшей Brainy Zombie с весом 10 во все разрешённые биомы.

## Giant Brainy Zombie

Восстановлены:

- здоровье `60`;
- базовый урон `7`;
- XP `15`;
- прыжок к цели с коэффициентом `0.4`;
- синхронизированная ярость `1.0–2.0`;
- рост ярости на `0.1` при получении урона;
- затухание на `0.002` за тик;
- кривая урона `7 + (anger - 1) × 5`, то есть `7–12`;
- collision scale `1.2 + anger`;
- renderer scale, равный текущей ярости;
- сохранение ярости в NBT `TC4Anger`;
- двенадцать независимых 50-процентных попыток дропа двух единиц гнилой плоти;
- та же формула дропа мозга;
- редкая ветка из thaumium ingot, carrot, potato и amber.

Для динамических размеров используется локальная безопасная копия ярости: она не читает `SynchedEntityData` во время выполнения superclass-конструктора, а после сетевого обновления вызывает `refreshDimensions()`.

Придуманное сопротивление отбрасыванию, присутствовавшее в раннем черновике стадии, удалено: оригинальный класс его не задавал.

## Регистрация и клиент

- Оба `EntityType` зарегистрированы существующим `DeferredRegister<EntityType<?>>`.
- Атрибуты подключены через `EntityAttributeCreationEvent` на mod event bus.
- Spawn placements регистрируются внутри `FMLCommonSetupEvent#enqueueWork` с `Monster.checkMonsterSpawnRules`.
- Добавлены `ForgeSpawnEggItem` с оригинальными цветами:
  - Brainy: `0xFFBFFF / 0x008000`;
  - Giant: `0xFFBFFF / 0x004000`.
- Для обеих форм используется оригинальная текстура `bzombie.png`, SHA-256:
  `802e3a399b6bd9466fb2282605a081635dd8e4fe2328604ee53ee94c90ac92c7`.
- Дальность отслеживания и update interval приведены к исходной регистрации: `64 / 3`.
- Добавлены английские и русские названия и item-модели яиц призыва.

## Сканирование и исследование

Используются ранее подготовленные контракты:

- `thaumcraft:brainy_zombie` → `Thaumcraft.BrainyZombie`;
- `thaumcraft:giant_brainy_zombie` → `Thaumcraft.GiantBrainyZombie`.

Aspect sets также совпадают с `ConfigAspects` оригинала. Оба legacy ID являются entity triggers скрытого исследования `JARBRAIN`.

## Осознанное отклонение

В оригинальном Eerie biome использовались отдельные веса:

- Brainy Zombie: `32`;
- Giant Brainy Zombie: `8`.

Полноценный Eerie biome в текущем порте ещё не завершён. Поэтому обычная форма уже появляется с глобальным весом 10, а гигант пока доступен через яйцо призыва/команды. Добавлять гиганта в Magical Forest было бы неоригинальной подменой.

## Gradle

`./gradlew compileJava --no-daemon --stacktrace` не дошёл до компилятора: Gradle Wrapper не смог разрешить `services.gradle.org` (`UnknownHostException`). В контейнере доступна только Java 21, тогда как Forge 1.19.x требует Java 17.

Следовательно, JAR этой стадии не заявляется как собранный, а API/renderer/runtime parity не помечаются как фактически проверенные в Minecraft.
