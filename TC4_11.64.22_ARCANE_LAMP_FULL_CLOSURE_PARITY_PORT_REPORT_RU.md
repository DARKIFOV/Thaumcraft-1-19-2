# TC4 v11.64.22 — Arcane Lamp: полное закрытие по исходникам и ресурсам

## Единственный объект раунда

Обычная **Arcane Lamp / Магический светильник** (`blockMetalDevice`, metadata 7), включая только её неразрывные пути:

- невидимый световой маркер `blockAiry`, metadata 3;
- прямую интеграцию с `TileArcaneBore` после успешной добычи;
- дополнительный соединительный nozzle при установке на `TileArcaneBoreBase`;
- рецепт, исследование, модель, текстуры, NBT и способы размещения.

`Lamp of Growth` и `Lamp of Fertility` в этот релиз не включались и закрытыми здесь не объявляются.

## Что было неверно

- Светильник требовал `isFaceSturdy`, хотя оригинал проверял только превращение опорного блока в воздух.
- Невидимый маркер ошибочно имел BlockEntity, source NBT и самоуничтожался без лампы.
- Кандидат Y зажимался к `maxBuildHeight - 1`; оригинал только ограничивал `surface + 4` и снизу Y=5, а выход выше современной границы должен отменять попытку.
- Пропускались незагруженные чанки через `hasChunkAt`, чего нет в исходном прямом world-path.
- Не сохранялся оригинальный NBT `orientation`.
- Отсутствовала установка света Arcane Bore после успешной добычи при наличии соседней Arcane Lamp.
- Корпус лампы вращался вместе с facing, из-за чего менялось назначение top/side текстур.
- Nozzle был приблизительно встроен в JSON и использовал текстуры лампы вместо `ModelBoreBase` + `Bore.png`.
- Не рендерился второй соединитель на стороне Arcane Bore Base.
- Рецепт потерял пробелы, Arcane Stone и правильное назначение Nitor/Glowstone.
- Item rarity была `UNCOMMON`, хотя оригинальный предмет использует обычную редкость.

## Исправленный production-код

- `TC4ArcaneLampParity` фиксирует формулы, границы, пороги и таблицу Arcane Bore без Forge-зависимостей.
- `ArcaneLampBlock` принимает любую не-воздушную опору; шесть направлений сохраняются.
- `ArcaneLampBlockEntity`:
  - одна попытка за серверный тик;
  - независимые разности двух `nextInt(16)` для X/Y/Z;
  - `min(sampledY, surface + 4)`, затем минимум Y=5/минимум измерения;
  - только воздух, не собственный marker, brightness `< 9`;
  - общий cleanup всех marker-блоков в кубе `[-15;15]`;
  - сохранение/загрузка `orientation` ordinal.
- `ArcaneLampLightBlock` теперь обычный `Block`, который считается воздухом, заменяем, невидим, не имеет коллизии и BlockEntity, даёт свет 15.
- Старый `ArcaneLampLightBlockEntity` оставлен только как deprecated migration registry carrier и не привязан к marker-блоку.
- `ArcaneBoreBlockEntity::mineTarget` вызывает production-путь установки marker:
  - соседняя лампа ищется NORTH, SOUTH, WEST, EAST;
  - `d = nextInt(32) * 2`;
  - фазы `+3 / 0 / -3 / 0`, а горизонтальная фаза 3 опускает Y на 2;
  - порог brightness `< 15`.
- `ArcaneLampRenderer` использует точные nozzle-кубы `ModelBoreBase` и `textures/models/Bore.png`, шесть исходных трансформаций и второй opposing connector на Arcane Bore Base.
- Корпус JSON оставлен одним фиксированным cuboid `4,2,4 → 12,14,12`; все facing-варианты ссылаются на него без вращений.
