# Аудит оставшихся TC4-ID — v11.63.37

Этот аудит отвечает на вопрос «сколько осталось» по воспроизводимому статическому критерию: ID всё ещё создаётся общим `TC4ResearchComponentItem`, а не отдельным классом или заранее зарегистрированным функциональным объектом.

| Показатель | Значение |
|---|---:|
| Записей в de-metadata mirror | 294 |
| ID с отдельной веткой `createItem` | 231 |
| Заранее зарегистрированные функциональные ID | 17 |
| Намеренно пропущенные дубли BlockItem | 4 |
| **Оставшиеся generic fallback ID** | **42** |
| Из них предметоподобные ID | 0 |
| Из них блочные migration-alias ID | 42 |

## Как читать число

**42 — это верхняя техническая оценка блочной миграции, а не число самостоятельных неперенесённых механик.** После v11.63.37 предметоподобных ID в общем fallback не осталось. Все текущие записи — старые блочные migration-alias; часть уже указывает на перенесённые системы и нуждается в нормализации регистрации/сохранений, а часть относится к ещё не закрытым блочным семействам.
