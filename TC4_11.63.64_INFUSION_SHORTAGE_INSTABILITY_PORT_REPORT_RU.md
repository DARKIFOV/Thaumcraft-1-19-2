# TC4 v11.63.64 — Нехватка эссенции/компонентов при инфузии: отчёт о портировании

## Как это было устроено в оригинале (Thaumcraft 4, 1.7.10, `TileInfusionMatrix.craftCycle`, строки ~415–490)

Когда активная инфузия временно не может продолжиться из-за нехватки ресурсов, оригинал **никогда не отменяет крафт** — он просто ждёт и по чуть-чуть накручивает дестабилизацию ("instability creep") каждый цикл, пока проблема не исчезнет:

1. **Нехватка эссенции.** Оригинал за один тик пытается слить **все** ожидающие (pending) аспекты подряд, а не только первый в списке. Как только хотя бы один аспект успешно слился — цикл прерывается (успех). Если ни один аспект не слился, для **каждого** неудачного аспекта отдельно бросается кубик `nextInt(100 - instability*3)`, и после полного прохода по всем аспектам матрица помечается флагом `checkSurroundings = true` (заставляет пересканировать пьедесталы/структуру на следующем цикле).
2. **Нехватка компонента.** Для **каждого** несовпавшего по позиции ингредиента (индекс `a` в списке компонентов) отдельно бросается `nextInt(1 + a)`. Если выпадает 0 — эссенция за этот компонент возвращается игроку/аспектам, и **внутри** этого же условия дополнительно накручивается дестабилизация `nextInt(50 - instability*2)`. Чем дальше по списку несовпавший компонент — тем меньше шанс сработать (знаменатель растёт с индексом).
3. **Нехватка XP/очков очарования** (recipeType == 1) — тот же паттерн 1-в-3 с вложенным броском дестабилизации; этот путь был перенесён верно ещё в прошлых раундах и не менялся.

## Что было неверно в порте до этого раунда

- **Нехватка эссенции** портировалась так, что матрица пробовала слить только **первый** аспект из списка ожидания и останавливалась на этом (даже если следующие аспекты были готовы слиться). Флаг `checkSurroundings` при полном провале не выставлялся — матрица не пересканировала пьедесталы после серии неудач.
- **Нехватка компонента** использовала общий (не зависящий от индекса ингредиента) шанс возврата эссенции, из-за чего все несовпавшие компоненты имели одинаковую вероятность триггера, а не убывающую по списку, как в оригинале.

## Что сделано в этом раунде (v11.63.64)

1. Переписан `InfusionMatrixBlockEntity.drainNextEssentia`: теперь метод итерируется по **копии** множества ожидающих аспектов (`new ArrayList<>(pendingAspects.keySet())`) в одном тике, пробуя слить каждый; при полном провале выставляется `checkSurroundings = true` перед откладыванием цикла.
2. Переписан `InfusionMatrixBlockEntity.pullNextComponent`: возврат эссенции по несовпавшему ингредиенту теперь гейтится через `TC4InfusionRuntime.componentShortageEssentiaRefundRollBound(i)` — новый метод, возвращающий `Math.max(1, 1 + ingredientIndex)`, то есть точную формулу `nextInt(1 + a)` по индексу.
3. Добавлен параллельный класс-контракт `TC4InfusionShortageInstabilityParity.java`, документирующий эти правила как источник истины (аналогично более ранним *Parity-контрактам этого порта).
4. Добавлено 5 новых GameTest-сценариев в `TC4BlockEntityGameTests.java` (66 → 71 тест):
   - нехватка эссенции/компонента никогда не отменяет крафт;
   - нехватка эссенции пробует каждый ожидающий аспект за цикл;
   - нехватка эссенции форсирует пересканирование (`checkSurroundings`), а нехватка компонента — нет;
   - первый несовпавший компонент всегда даёт шанс на возврат эссенции;
   - шанс возврата падает с ростом индекса несовпавшего компонента.
5. Версия обновлена до `11.63.64` (build.gradle, mods.toml — со старыми версиями сохранёнными как markers-комментарии для совместимости старых guard'ов).
6. Манифест раннтайм-тестов обновлён до 399 сценариев (394 → 399).
7. Создан новый статический guard `tc4_116364_infusion_shortage_instability_guard.py`, проверяющий формулы, GameTest-методы и манифест — PASS.
8. **Исправлены 13 более старых guard'ов** (`tc4_116350` … `tc4_116363`), которые сверяют точное/накопительное число GameTest'ов и версию манифеста — они не принимали новые значения (71 тест, 399 сценариев, версия 11.63.64) и реально проваливались. Теперь все они дополнены соответствующими допустимыми значениями и проходят.
9. Обновлена и прогнана полная CI-цепочка (`run_full_static_ci_116364.py`, 130 проверок). Новый и связанные с этим раундом guard'ы — PASS. Оставшиеся падения (диапазон ~11.63.78–11.63.49) — это давно известный, не связанный с текущими изменениями дрейф версии в старых legacy-проверках, существовавший и до этого раунда.

## Ограничения

- В сэндбоксе нет доступа к сети (`services.gradle.org` недоступен), поэтому полноценная Gradle-компиляция (`./gradlew build`) не может быть запущена здесь — верификация ограничена статическими guard'ами (структура кода, точные формулы, наличие методов, целостность манифеста), а не байткодом/раннтаймом Minecraft.

## Что осталось для полного портирования

В манифесте и общем статусе `TC4_PORT_STATUS_V3.md` отслеживается остаточный список неперенесённых механик Thaumcraft 4 (по секциям crafting/blocks/entities/world-gen); нехватка эссенции/компонентов при инфузии закрыта этим раундом. Продолжение — по следующим пунктам списка нерешённых контрактов в `TC4_PORT_STATUS_V3.md`.
