## 3. Завершение употребления еды

Сохранён оригинальный post-use порядок:

- rotten flesh или `thaumcraft:tc4_brain` уменьшают оставшуюся длительность ровно на 600 тиков и amplifier ровно на 1;
- текущий effect instance снимается;
- новый instance возвращается только при `duration > 0 && amplifier >= 0`;
- у возвращённого эффекта explicit curative list содержит только rotten flesh, как в TC4;
- специальная еда выводит `warp.text.hunger.2` тёмно-зелёным курсивом;
- обычная еда выводит `warp.text.hunger.1` тёмно-красным курсивом;
- питание обычной еды не отменяется.

## 4. Клиентский визуал

Предыдущая временная зелёная полупрозрачная HUD-заливка удалена. Вместо неё добавлен отдельный `PostChain`:

`assets/thaumcraft/shaders/post/unnatural_hunger.json`

Он воспроизводит параметры оригинального `hunger.json`:

- `ColorScale = [1.0, 0.8, 0.8]`;
- `Saturation = 1.1`;
- первый проход `color_convolve` из `minecraft:main` в `swap`;
- второй проход `blit` обратно в `minecraft:main`.

`UnnaturalHungerPostEffect`:

- обрабатывает эффект перед полным HUD;
- владеет отдельной post-chain цепочкой и не занимает единый глобальный post-effect slot;
- пересоздаёт/resize цепочку при изменении framebuffer;
- возвращает main framebuffer после обработки;
- закрывает ресурсы при снятии эффекта или отсутствии игрока;
- инвалидирует цепочку при client resource reload;
- не спамит повторной ошибкой загрузки до следующего reload.

Поэтому отсутствуют:

- собранный Forge JAR;
- запуск dedicated GameTest server и фактический результат 146 тестов;
- клиентский кадр exact post-effect;
- проверки resize/resource reload;
- save/reload эффекта;
- два клиента, latency/reconnect и совместимость с другими post-effects.

Итоговая формулировка: **все найденные расхождения Unnatural Hunger в production source/resources закрыты и защищены статическими контрактами; окончательное 100% игровое закрытие ожидает реальной Java 17 Forge-сборки и runtime-протокола.**
