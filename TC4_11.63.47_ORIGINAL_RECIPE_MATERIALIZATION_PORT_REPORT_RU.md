| Вход | Выход | Количество | XP | Время |

| Iron Ore Cluster | Iron Ingot | 2 | 1.0 | 200 |
| Cinnabar Ore Cluster | Quicksilver | 2 | 1.0 | 200 |

Исправлены прежние отклонения:

- кластеры и Balanced Shard выдавали `0.1 XP` вместо `1.0`;
- монета ошибочно выдавала опыт вместо `0.0`;
- Balanced Shard использовал не exact mirror-ID;
- Cinnabar Cluster выводил канонический предмет вместо точного migration-ID `tc4_quicksilver`.

Для двухпредметного результата используется `CountedSmeltingRecipeSerializer`, который передаёт полный `ItemStack` по сети и сохраняет количество, XP и время приготовления.

## Infusion enchantment

Все **24** оригинальные записи включены в отдельное runtime-семейство:

- 22 vanilla-enchantment;
- 2 Thaumcraft-enchantment: Repair и Haste.

Сохранены формулы из `InfusionEnchantmentRecipe.java`:

- базовый XP: `max(1, minCost(1) / 3)`;
- итоговый XP: `baseXP × (1 + текущий уровень целевого зачарования)`;
- нестабильность: базовая нестабильность + целая половина суммы существующих уровней;
- essentia-модификатор: текущий уровень цели + `0.1 ×` уровни остальных зачарований;
- результат повышает целевое зачарование на один уровень, не удаляя остальные;
- несовместимые и уже максимальные зачарования отклоняются.

Матрица выполняет проверку, масштабирование essentia, списание XP и применение результата через единый adapter. Историческая запись «serializer отсутствует» больше не отражает фактическое состояние runtime; новый ledger фиксирует это отдельно, не переписывая старый source-аудит.

## GameTest

Добавлены два обязательных server-only GameTest:

1. `originalSmeltingRecipesMatchTC4` — загружает пять рецептов через живой `RecipeManager` и проверяет вход, выход, количество, XP и 200 тиков.
2. `infusionEnchantmentFamilyMatchesTC4` — проверяет наличие 24 записей, компоненты Protection, формулы XP/нестабильности/essentia, повышение Protection I → II, сохранение Unbreaking III и отклонение максимального уровня.

Общий обязательный пакет вырос до **12 GameTests**, runtime-манифест — до **330 сценариев**. Все новые runtime-сценарии имеют статус `NOT_TESTED`, пока не выполнен настоящий Java 17 build и `runGameTestServer`.
