# TC4 v11.64.34 — Infusion Altar full-closure report

## Единственная закрываемая механика

**Infusion Matrix, Arcane Pedestal, четыре infusion pillars и полный жизненный цикл инфузии.**

В этот релиз не включались essentia-трубы, банки и алхимические машины как отдельные системы. Матрица использует их существующий общий интерфейс только как внешний источник essentia.

## Найденные оригинальные пути

Проверены `WandManager#createInfusionAltar`, `fitInfusionAltar`, `TileInfusionMatrix`, `TilePedestal`, `TileInfusionPillar`, `BlockStoneDevice`, renderers матрицы/пьедестала/колонн, обычные и enchantment infusion-рецепты, сохранение NBT, comparator, block events, контейнерные предметы и instability event table.

## Исправления production-кода

- Возвращено создание алтаря жезлом только после исследования `INFUSION`.
- Поиск исходной точки выполняется в диапазоне clicked−2…clicked по трём осям.
- Проверяется точный пустой чертёж 3×3×3: matrix, центральный pedestal, 4 arcane stone, 4 arcane stone bricks и 17 air.
- Списание 25 vis каждого из шести первичных аспектов выполняется атомарно с учётом скидки жезла.
- Удалён дублирующий путь создания алтаря.
- Четыре угла превращаются в двухблочные pillars с исходными orientation 2/3/4/5.
- Нижняя половина возвращает arcane stone bricks, верхняя — arcane stone; разрушение удаляет связанную половину.
- Pillar скрыт из creative-каталога как внутренний преобразованный блок.
- Пьедестал восстановлен как один слот со всех сторон, лимитом 1, comparator 0/15, `Items/Slot` и `CustomName` NBT.
- Восстановлены оригинальные звуки помещения/снятия, выпадение предмета и block events 11/12.
- Renderer пьедестала использует y=1.15, bob `sin(t/16)*0.05`, 1°/tick, scale 2 для блоков и второй плоский render-проход.
- Matrix light level — 10; разрушение работающей матрицы вызывает explosion 2.0.
- `validLocation` проверяет центральный pedestal и четыре нижние колонны.
- Pedestal scan — x/z ±8, y от −1 до −10, только первый pedestal в каждом x/z-столбце.
- Stabilizer scan — x/z ±12, y +5…−10.
- Validity cadence: 100 тиков в ожидании и 20 при crafting.
- Normal craftCycle — каждые 10 тиков; XP-stage — каждые 20.
- Рецепт, catalyst, список компонентов, аспекты, instability, XP и владелец блокируются на старте.
- Нет обязательной preflight-проверки всей essentia: работающая матрица ждёт поступления каждого аспекта.
- Essentia расходуется по одной единице; после полного неудачного прохода включается повторный scan.
- Компоненты забираются в исходном порядке; item travel длится пять последующих craftCycle, container item сохраняется.
- Восстановлена полная таблица из 21 instability event.
- Стартовая instability равна `symmetry + recipeInstability` без предварительного clamp.
- Gate использует точное `instability > 0 && nextInt(500) <= instability`.
- Только shortage-increment ограничивает верхнее значение 25; повторный scan не переписывает уже запущенную instability.
- Удалено влияние неоригинального accelerator на cadence production craftCycle.
- Завершение поддерживает обычный ItemStack, NBT-output, infusion enchantment и runic augment.
- Сохранение использует оригинальные поля `active`, `crafting`, `instability`, `Aspects`, `recipein`, `rotype`, `recipeout`, `recipeinput`, `recipeinst`, `recipetype`, `recipexp`, `recipeplayer`; `recipeid` оставлен только как минимальный registry bridge Forge 1.19.2.
- Незавершённая инфузия после reload сохраняет остатки аспектов/компонентов и продолжает цикл; transient counters и source FX не сохраняются.
- Оригинальные pillar/matrix/pedestal textures проверены по SHA-256; geometry pillar OBJ совпадает после удаления только Forge MTL-директив.
