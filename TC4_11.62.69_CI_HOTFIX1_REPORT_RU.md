# Thaumcraft Legacy Rebuild 11.62.69 — CI HOTFIX 1

## Причина сбоя

В загруженном логе GitHub Actions сборка останавливалась на шаге:

`Validate magical leaves, wand UV and Research Table renderer`

Guard обнаруживал, что `WandItemRenderer` потерял три обязательных элемента адаптера оригинальной геометрии `ModelWand`:

- зеркальное смещение для левой и правой руки: `poseStack.translate(left ? -0.11D : 0.11D, 0.06D, 0.02D)`;
- константу `MODEL_CENTER_Y`;
- центрирование модели: `poseStack.translate(0.0D, -MODEL_CENTER_Y, 0.0D)`.

Из-за этого GitHub Actions завершался с кодом 1 до запуска Gradle.

## Исправления

### 1. Рендер жезла

В `WandItemRenderer.java` восстановлен адаптер исходной модели TC4:

- добавлен центр геометрии `MODEL_CENTER_Y = 0.625D`;
- модель центрируется относительно локального начала координат BEWLR;
- добавлено зеркальное смещение для левой и правой руки;
- сохранён отдельный масштаб first-person режима.

Это исправляет контракт guard и предотвращает смещение жезла за рукой либо его пересечение с камерой.

### 2. Модель Arcane Workbench в исходном пакете

Поставленный исходный архив содержал устаревший вариант `arcane_workbench.json`, не соответствовавший уже существующему screenshot guard. Модель синхронизирована с текущим контрактом:

- сохранены шесть элементов геометрии;
- верх использует `arcane_workbench_top`;
- боковые поверхности используют `arcane_workbench_side`;
- восстановлен корректный particle texture.

### 3. Устаревший BEWLR-аудит

`tc4_116246_audit_guard.py` всё ещё требовал ровно восемь `builtin/entity`-моделей, хотя после портирования Hungry Chest, Banner и Vis Charge Relay их стало одиннадцать.

Guard обновлён так, чтобы проверять точный ожидаемый набор из 11 моделей, а не устаревшее числовое значение. Теперь он выявляет как отсутствующие, так и неожиданные регистрации.

### 4. Ошибка генератора исследований

Обнаружен следующий скрытый CI-блокер: генератор `generate_tc4_research_runtime_from_source_map.py` создавал синтаксически неверный Java-код для исследований без страниц, например:

```java
put(map, "CAP_iron",
```

Теперь для записей с нулём страниц генерируется корректный вызов:

```java
put(map, "CAP_iron");
```

После исправления повторная генерация полностью совпадает с сохранёнными Java-индексами, а `--check` проходит.
