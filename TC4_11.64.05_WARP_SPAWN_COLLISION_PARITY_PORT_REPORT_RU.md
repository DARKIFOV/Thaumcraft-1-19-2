# Thaumcraft Legacy Rebuild v11.64.05 — Warp spawn entity-aware collision parity

## Цель раунда

Исправить серверный поиск точки появления Eldritch Guardian и Mind Spider при warp-событиях по фактическому контракту TC4 4.2.3.5.

## Подтверждённое расхождение

Оригинальный `WarpEvents.java` для каждой из 50 попыток:

1. выбирает три смещения `random(7,24) * random(-1,1)`;
2. требует твёрдую верхнюю поверхность блока под точкой;
3. перемещает конкретную сущность в кандидатную точку;
4. проверяет её реальный bounding box на коллизии;
5. отклоняет bounding box, пересекающий жидкость.

Оригинал не требует фиксированную колонну из двух воздушных блоков. В порте общий helper заранее требовал `pos.isAir()` и `pos.above().isAir()` для обеих сущностей. Это было слишком строго для `MindSpiderEntity` высотой 0.3 блока и отбрасывало низкие, но фактически допустимые места.

## Production-исправление

`WarpEvents.findSpawnAround` теперь:

- сохраняет 50 попыток и исходные tri-state offsets `-1/0/+1`;
- проверяет твёрдую верхнюю поверхность блока под кандидатом;
- сначала перемещает конкретную сущность;
- использует `level.noCollision(entity)` для её реального bounding box;
- сканирует только блоки, пересекаемые bounding box, на наличие жидкости;
- принимает решение через `TC4WarpRuntimeParity.acceptsEntitySpawnCandidate`;
- больше не требует `pos` и `pos.above` быть воздухом заранее.

Высокий Eldritch Guardian по-прежнему отклоняется в низком пространстве самой collision-проверкой, а низкий Mind Spider может использовать пространство, в которое реально помещается.

## Новые GameTest-контракты

Добавлены два обязательных server-only GameTest-метода:

1. `warpSpawnCandidateUsesOriginalCollisionLiquidContract` — таблица допуска: solid top + no collision + no liquid.
2. `warpSpawnCandidateUsesActualEntityDimensions` — Mind Spider высотой 0.3 помещается под одно-блочным потолком, Guardian высотой 2.2 там отклоняется.

Эти методы существуют в исходниках, но не считаются runtime PASS без запуска Forge GameTest server.
