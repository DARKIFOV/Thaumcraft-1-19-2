## Найденные расхождения v11.64.15

1. Для outline и collision использовалась одна форма высотой 14/16. В оригинале outline имеет высоту 14/16, а физическая коллизия — 15/16.
2. `entityInside` принимал любой предмет, попавший в координатный куб блока, без явной проверки исходного inset collision box.
3. Сервер каждые 80 тиков пересчитывал открывших игроков. В финальном декомпилированном TC4 это условие пустое; состояние меняется только через `openInventory`/`closeInventory` и block event 1.
4. Меню закрывалось на расстоянии более восьми блоков. Оригинальный `isUseableByPlayer` проверяет только, что в позиции всё ещё находится тот же tile entity.
5. Opener count и event data искусственно ограничивались нулём, хотя оригинал напрямую присваивает/увеличивает/уменьшает значения.
6. При разрушении использовался `Containers.dropContents`. Оригинал для каждого слота выбирает одну позицию и делит стак на части `nextInt(21)+10`, сохраняя NBT и задавая Gaussian-скорость.
7. Item-renderer содержал приблизительные отдельные трансформации для GUI, рук, земли и рамки. Оригинал использует один путь: поворот Y на 90° и перенос `(-0.5,-0.5,-0.5)` перед TESR.
8. Взрывоустойчивость была задана как 4.0. Старый `setHardness(2.5F)` автоматически поднимал resistance до `hardness × 5 = 12.5`.

## Production-исправления

### Блок и геометрия

- Outline: `(1/16, 0, 1/16) → (15/16, 14/16, 15/16)`.
- Collision: `(1/16, 0, 1/16) → (15/16, 15/16, 15/16)`.
- Поглощение разрешено только при реальном пересечении bounding box предмета с исходной collision-формой.
- Сохранены четыре горизонтальных направления, `ENTITYBLOCK_ANIMATED`, отсутствие occlusion и запрет pathfinding.
- Hardness 2.5 и explosion resistance 12.5 закреплены единым parity-контрактом.

### Инвентарь и поглощение

- Ровно 27 доступных слотов и лимит стака 64.
- Один реальный контейнер используется меню, comparator и Forge item capability со всех сторон.
- Строго совместимые существующие стаки заполняются раньше пустых слотов.
- Полный приём удаляет ItemEntity; частичный сохраняет точный остаток; полный отказ ничего не меняет.
- Успешное перемещение один раз проигрывает `GENERIC_EAT` с громкостью 0.25 и формулой pitch `(r1-r2)*0.2+1.0`.
- После успешного перемещения посылается block event `(2,2)` и крышка поднимается минимум до 0.2.

### Крышка, меню и сеть

- Шаг крышки 0.1 за тик, диапазон 0..1.
- Звук открытия только из полностью закрытого состояния.
- Звук закрытия при пересечении 0.5 вниз.
- Pitch сундука: `random*0.1+0.9`, громкость 0.5.
- Opener count синхронизируется только block event 1; eat nudge — block event 2.
- Удалён придуманный периодический recount.
- `stillValid` проверяет только идентичность block entity, как TC4.
- Opener count и lid state не сохраняются в NBT.

### Разрушение

- Для каждого непустого слота один раз выбираются X/Y/Z в диапазоне `0.1..0.9`.
- Содержимое делится на части 10–30 предметов, последняя часть может быть меньше 10.
- Все части одного исходного слота появляются в одной позиции.
- Скорость: Gaussian `0.05`, вертикальная добавка `+0.2`.
- Сохраняются custom name и полный NBT исходного ItemStack.

### Клиент и ресурсы

- Используется точная ванильная ModelChest-геометрия 1.7.10 и исходная 64×64 UV-развёртка.
- `chesthungry.png` побайтово совпадает с сохранённым оригиналом.
- Сохранено cubic easing `1-(1-angle)^3` и исходные повороты metadata 2–5.
- BEWLR использует единый оригинальный inventory transform без приблизительных context-specific поправок.
- Particle icon блока остаётся исходным `woodplain`.

### Рецепт, исследование, creative и JEI

- Arcane pattern: `WTW / W W / WWW`.
- `W = #minecraft:planks`, `T = minecraft:oak_trapdoor`.
- Стоимость: Aer 5, Ordo 3, Perditio 3.
- Исследование HUNGRYCHEST: GOLEMANCY `(-1,0)`, complexity 1, Fames 3, Vacuos 3, secondary, две исходные страницы.
- GOLEMSTRAW сохраняет HUNGRYCHEST как родителя.
- Предмет остаётся в Thaumcraft creative tab; рецепт входит в общий Arcane Workbench/JEI data path.
- Hungry Chest остаётся точным катализатором инфузии Traveling Trunk.
