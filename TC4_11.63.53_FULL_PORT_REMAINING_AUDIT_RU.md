# TC4 v11.63.53 — остаток до полного порта Thaumcraft 4.2.3.5

## Что уже закрыто

На уровне данных и исходных контрактов подтверждены:

- 201 исследований и 591 страница;
- 48 аспектов;
- original recipe-ledger **258/258**;
- generic registry fallback **0**;
- удаление 63 доказанных item alias-ID с каноническим remap;
- 663 канонические item-модели без missing/unresolved ресурсов;
- 24 обязательных server-only GameTest-контракта;
- 352 сценария в runtime-манифесте;
- отсутствие прямой зависимости от внутренних mixin-классов FancyMenu.

Это не равно полному runtime-паритету. До заявления «полный порт оригинала» остаются следующие доказательства.

## P0 — обязательные блокеры релиза

### 1. Точная Java 17 сборка v11.63.53

Нужно получить одним воспроизводимым прогоном:

- `gradlew clean build`;
- release JAR;
- SHA-256 JAR;
- `gradlew runGameTestServer`;
- **24/24 GameTests PASS**;
- чистый старт dedicated server и клиента именно с этим JAR.

Текущая среда остановилась на DNS до конфигурации Gradle. Старый 18/18 PASS версии 11.63.51 не переносится на изменённую 11.63.53.

### 2. Fixture-миграция настоящих миров

Нужны сохранения старых версий с каждым из 63 удалённых alias-ID и raw aura nodes в:

- player inventory, armor и Ender Chest;
- dropped item entities;
- обычных контейнерах;
- Hungry Chest и Traveling Trunk;
- nested Forge item handlers/capabilities;
- машинных слотах и serialized ItemStack NBT.

Нужно проверить два последовательных save/reload, count, damage, name, enchantments, NBT и capabilities.

### 3. Клиентский GUI и визуальный протокол

Нужны реальные screenshot/runtime проверки:

- 201 research entries и 591 страницы при нескольких GUI Scale;
- Thaumonomicon tree и hidden/concealed progression;
- Research Table;
- Thaumometer scan overlay;
- Arcane Workbench, infusion, essentia containers и machines;
- wand/focus HUD, beams, nodes и first/third-person transforms;
- обычный инвентарь и все mod containers;
- отдельная compatibility-матрица FancyMenu и других screen modifiers без hard dependency.

### 4. Multiplayer/network

Нужен dedicated server с двумя клиентами:

- research, aspects, warp и vis sync;
- одновременная работа с контейнерами и infusion;
- wand/focus actions и FX packets;
- golem markers/upgrades;
- mirrors и dimension travel;
- reconnect, death и save/reload без рассинхронизации.

### 5. Outer Lands end-to-end

Нужно доказать полный игровой цикл:

- портал и dungeon allocation;
- достижимый maze;
- locks, keys и progression;
- traps, loot, mobs и bosses;
- return portal;
- death/re-entry;
- несколько игроков и несколько dungeon instances;
- save/reload и chunk unload.

## P1 — крупные системы полного оригинала

### 6. Полная Golemancy matrix

Проверить все тела, cores, upgrades, accessories, markers, filters, NBT routing, pathfinding, chunk boundaries и automation races. Текущий GameTest Traveling Trunk не заменяет полный golem automation protocol.

### 7. Essentia и alchemy long-soak

Нужна матрица всех 48 аспектов для furnace, alembics, tubes, valves, filters, buffers, reservoirs, jars, centrifuge и Thaumatorium. Обязательны suction conflicts, blocked outputs, rollback, unload/reload и длительные server runs.

### 8. Infusion runtime

Нужно проверить pedestal discovery, symmetry, stabilizers, instability events, item loss, flux/explosions, insufficient essentia, stop/resume, central-stack NBT copy, multiplayer ownership и все 63 infusion + 24 enchantment source records.

### 9. Research, warp и Eldritch progression

Требуются end-to-end playthrough tests от стартовых исследований до concealed/warp-gated веток, Warp Ward, random warp events, Crimson progression, eldritch portal chain, death/reconnect и multiplayer sharing.

### 10. Nodes, taint и worldgen

Нужны длительные fixture worlds для node types/modifiers, recharge/drain, stabilizer/transducer/relay, taint ecology, Magical Forest/Eerie/Tainted biomes, trees, ores, structures, Mana Pods, config toggles и server performance.

## P2 — выпуск и область проекта

### 11. Core против addon/compat

Нужно окончательно выделить core-only профиль либо документировать compatibility modules. Похожие предметы нельзя удалять без доказательства registry-эквивалентности и безопасной миграции миров.

### 12. Release engineering

Перед релизом нужны:

- воспроизводимая Java 17 CI;
- release JAR и checksums;
- migration guide/changelog;
- лицензии и разрешения на assets/code;
- known incompatibilities;
- clean-install tests без dev/source JAR;
- client и dedicated-server soak tests.
