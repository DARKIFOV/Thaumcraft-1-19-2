# Thaumcraft Legacy Rebuild 1.19.2 — v11.63.29

## Стадия

**Simple Resource Families / Простые ресурсные семейства** — первая пакетная стадия после перехода от уникальных механизмов к массовому закрытию пассивных предметов TC4.

## Закрытые ID

Всего материализовано **33 ID**, ранее создававшихся общим `TC4ResearchComponentItem`:

- 12 самородков: 8 металлических и 4 съедобных мясных;
- 8 осколков: шесть первичных, сбалансированный и нейтральный migration-ID;
- 7 рудных кластеров;
- 6 базовых материалов: янтарь, ртуть, магическое сало, зачарованная ткань, таумиевый слиток и слиток металла Пустоты.

## Реализовано

### Самородки

- Металлические варианты используют отдельный `TC4SimpleResourceItem` с явным типом `METAL_NUGGET` и material key.
- Beef/Chicken/Fish/Pork используют `TC4EdibleNuggetItem`.
- Восстановлены параметры `ItemNuggetEdible`: 1 единица голода, saturation modifier `0.3`, время употребления 10 тиков.
- Добавлены `forge:nuggets/<material>` для iron, copper, tin, silver, lead, quicksilver, thaumium и void_metal, а также агрегирующий `forge:nuggets`.

### Осколки

- Шесть первичных и balanced-вариант подключены к существующему функциональному `ShardItem`, сохраняя aspect resonance и исследовательский insight.
- Legacy-ID добавлены в Forge/Thaumcraft shard tags.
- Нейтральный `tc4_shard` сохранён как отдельный migration-носитель без выдуманного аспекта.

### Рудные кластеры

- Семь вариантов используют `TC4OreClusterItem` с явным material key.
- Добавлены `forge:clusters/<material>` и агрегирующий `forge:clusters`.
- Сохранены существующие counted-smelting рецепты для железа, золота, меди и киновари. Tin/Silver/Lead остаются совместимыми через material tags и требуют установленного источника соответствующего слитка для data-driven рецепта.

### Базовые материалы

- Legacy thaumium/void ingots включены в соответствующие Forge ingot tags.
- Добавлены compatibility tags для amber, quicksilver, tallow и enchanted fabric.
- Void Metal Ingot помечен fire-resistant.
- Удалена автоматическая миграция этих закрытых ID в параллельные rebuild-ID: исходные TC4 registry ID теперь остаются самостоятельными функциональными предметами и не меняются при входе игрока.

## Исправление аудита остатка

Старый аудит считывал только первый ID из объединённых Java `case "a", "b" ->`. Поэтому число 188 было завышено на один уже закрытый grouped-ID. После исправления парсера корректная база v11.63.28 составляет 187, а после удаления 33 ресурсов остаётся **154 fallback-ID**: 112 предметных и 42 блочных migration aliases.

## Runtime и сборка

Пищевые параметры, tags, smelting, recipe viewers, save/reload и отсутствие миграции ID требуют проверки в запущенном Minecraft.

В доступной среде найден только OpenJDK 21.0.10; JDK 17 отсутствует. После проверки Wrapper команда `./gradlew --version` остановилась до запуска Gradle с `UnknownHostException: services.gradle.org`. Поэтому `compileJava` и `clean build` под неподходящей Java 21 не запускались, строки `BUILD SUCCESSFUL` нет, новый JAR v11.63.29 не создан.
