# Thaumcraft Legacy Rebuild Forge 1.19.2 — v11.63.44
## Обязательный headless Forge GameTest smoke-пакет

## Цель этапа

После закрытия registry fallback и добавления отложенной миграции старых миров главным риском оставалось отсутствие воспроизводимого runtime-доказательства для наиболее сложных persistent-систем. Ручной список `NOT_TESTED` не обнаруживает ошибки загрузки `BlockEntity`, NBT, тиков, registry wiring и серверных взаимодействий.

Этап 11.63.44 добавляет настоящий Forge GameTest-слой, который должен запускаться на отдельном headless `gameTestServer` после успешной Java 17 компиляции. Он не заменяет клиентские визуальные тесты, multiplayer и старые мировые fixture, но создаёт первый автоматический runtime gate.

## Архитектура

### Регистрация

Добавлен класс:

`src/main/java/com/darkifov/thaumcraft/gametest/TC4BlockEntityGameTests.java`

Он:

- регистрируется через `RegisterGameTestsEvent` на mod event bus;
- использует `@PrefixGameTestTemplate(false)`;
- задаёт `templateNamespace = "thaumcraft"` для каждого метода;
- не импортирует клиентские классы;
- содержит ровно шесть обязательных `@GameTest`-методов;
- завершает каждый сценарий только через `GameTestHelper#succeed`; нарушение контракта выбрасывает `GameTestAssertException`.

### Structure template

Добавлен пустой шаблон:

`data/thaumcraft/structures/empty_9x5x9.nbt`

Характеристики:

- gzip NBT;
- Minecraft DataVersion 3120;
- размер 9×5×9;
- воздушная palette;
- без встроенных block/entity fixture.

Все реальные блоки и `BlockEntity` создаются кодом теста, поэтому один шаблон не дублирует их сериализованные состояния и проще поддерживается при следующих изменениях.

### Gradle/CI

`gameTestServer` настроен с:

- `forge.enabledGameTestNamespaces=thaumcraft`;
- `sourceSets.main`;
- `forceExit false` для корректного завершения ForgeGradle 5;
- отдельным working directory `run-gametest`.

Оба GitHub Actions workflow теперь:

1. выполняют все source/resource guards;
2. запускают `gradlew build`;
3. только после успешной сборки запускают `gradlew runGameTestServer`;
4. сохраняют `gradle-build.log` и `gametest-server.log`;
5. завершают job ошибкой при падении сборки или любого обязательного GameTest;
6. только после этого вычисляют SHA-256 JAR и обновляют объективный статус.

## Шесть обязательных GameTests

### 1. Essentia Jar NBT round-trip

`essentiaJarPersistsFilterAndContents`

Проверяет:

- реальную регистрацию блока и `BlockEntity`;
- фильтр Aer;
- приём 48 Aer;
- отказ принять Ignis;
- сохранение фильтра, аспекта и количества через `saveWithoutMetadata`/`load`.

### 2. Wand Pedestal + Aura Node

`wandPedestalDrainsNodeAndChargesWand`

Проверяет:

- размещение пьедестала и живого узла;
- поиск узла серверным кодом;
- пять серверных тиков пьедестала;
- удаление ровно одной единицы Aer;
- начисление ровно 100 centivis жезлу;
- запись позиции источника для wispy-визуала.

### 3. Node Jar profile round-trip

`nodeJarPersistsProfileAndCaptureAnimation`

Проверяет:

- сохранение заполненного профиля банки;
- постоянный `NodeId`;
- текущие аспекты;
- начальный масштаб анимации захвата 3.0;
- восстановление через NBT.

### 4. Mana Pod growth/aspect round-trip

`manaPodGrowsAndPersistsAspect`

Проверяет:

- поддержку под бревном;
- рост стадии 2 → 3;
- сохранение единственного родительского аспекта Ignis;
- отсутствие сканируемого аспекта до зрелости;
- один аспект на стадии 7;
- NBT round-trip `ManaPodBlockEntity`.

### 5. Legacy ItemStack migration payload

`legacyStackMigrationPreservesPayload`

Проверяет:

- существование registry-ID `tc4_nitor`;
- замену на canonical Nitor;
- сохранение количества;
- сохранение custom name;
- сохранение пользовательского NBT.

### 6. Arcane Door master/access delegation

`arcaneDoorUpperHalfDelegatesPersistentAccess`

Проверяет:

- реальные нижнюю и верхнюю половины двери;
- UUID владельца;
- standard/full access lists;
- делегирование верхней половины к persistent нижней;
- правильную позицию привязки ключа.

## Ограничения и честный статус

В текущем окружении GameTests фактически не запускались. Gradle Wrapper не смог скачать Gradle 7.5.1 из-за `UnknownHostException: services.gradle.org`, а установленный runtime — OpenJDK 21.0.10 вместо требуемого Java 17.

Поэтому не заявлены:

- успешный `gradlew build`;
- успешный `runGameTestServer`;
- подтверждённый JAR;
- GameTest PASS;
- dedicated-server или multiplayer PASS;
- визуальная parity.
