# Отчёт о раунде портирования v11.64.10 — точная геометрия Death Gaze

Minecraft 1.19.2 · Forge 43.5.2 · целевая Java 17 · эталон Thaumcraft 4.2.3.5 (1.7.10)

## Найденное расхождение

Оригинальный `WarpEvents.checkDeathGaze` передавал в `EntityUtils.isVisibleTo` значение `0.75F`. Это значение не являлось готовым порогом dot product. В оригинальном `Utils.isLyingInCone` оно трактовалось как **полная апертура конуса в радианах**:

1. вычислялась половинная апертура `0.75 / 2`;
2. нормализованный dot product сравнивался строго с `cos(0.75 / 2)`;
3. цель ограничивалась круглой торцевой плоскостью по осевой проекции `< range`;
4. точкой цели служил центр тела: `boundingBox.minY + height / 2`;
5. до геометрии требовался `Entity.canBeCollidedWith()`.

В порте v11.64.09 использовалось другое поведение:

- прямое сравнение `dot >= 0.75`, дававшее почти вдвое более широкий конус;
- точка глаз цели вместо центра тела;
- отдельный сферический фильтр `distance <= range`, которого в исходном алгоритме не было;
- отсутствовал эквивалент исходного `canBeCollidedWith` admission gate.

Практически это позволяло Death Gaze поражать заметно более боковые цели и одновременно ошибочно отбрасывало некоторые цели у края круглой торцевой плоскости конуса.

## Исправление production-кода

### `TC4WarpRuntimeParity`

Добавлен source-linked чистый метод `deathGazeConeContains(...)`:

- `DEATH_GAZE_APERTURE_RADIANS = 0.75D`;
- строгое сравнение с `Math.cos(DEATH_GAZE_APERTURE_RADIANS / 2.0D)`;
- осевая проекция вычисляется отдельно;
- дальняя граница — строго `axialProjection < range`;
- вырожденные векторы и неположительный range отклоняются.

### `WarpEvents.checkDeathGaze`

- удалён `distanceToSqr <= range²` из предварительного фильтра;
- восстановлен `target.isPickable()` как современный эквивалент `Entity.canBeCollidedWith()`;
- точка цели изменена с `target.getEyePosition()` на центр тела;
- production делегирует всю геометрию в `TC4WarpRuntimeParity.deathGazeConeContains`.

Line-of-sight, PVP-ограничение, отсутствие уже наложенного Wither, запись атакующего и агр моба сохранены.

## Проверки

Добавлен GameTest `deathGazeUsesTc4ApertureBodyCenterAndRoundCap`, проверяющий:

- цель на оси;
- цель под углом 20° внутри оригинального конуса;
- цель под углом 30°, которую старый `dot >= 0.75` ошибочно принимал;
- точку с евклидовой дистанцией больше range, но допустимой осевой проекцией;
- строгую дальнюю границу `< range`;
- отклонение цели позади игрока.

Добавлены:

- manifest-сценарий `gametest.death_gaze_tc4_cone_geometry`;
- guard `tools/tc4_116410_death_gaze_cone_guard.py`;

Старый накопительный guard `tc4_116281_warp_eldritch_parity_guard.py` исправлен: он больше не требует ошибочный литерал `>= 0.75D`, а проверяет production-связь с точным новым алгоритмом.
