# Thaumcraft Legacy Rebuild v11.63.03 — исправление визуала и входа в Outer Lands

## Диагностика по присланным скриншотам

1. **Скипетр перекрывал экран от первого лица.** В современном
   `BlockEntityWithoutLevelRenderer` повторно использовалась матрица старого
   TC4 `IItemRenderer.EQUIPPED_FIRST_PERSON`: смещение по Y и масштаб
   накладывались поверх уже готового hand-space Minecraft 1.19.2.
2. **Преобразователь узла выглядел как сплошные чёрно-малиновые пластины.**
   Светящийся слой принудительно рисовался с `FULL_BRIGHT` и меняющимся alpha
   через cutout-слой. Alpha там не воспроизводил старый lightmap TC4 и визуально
   заливал детали модели.
3. **Предмет преобразователя отличался от установленного блока.** Для item-view
   использовалась отдельная кубическая JSON-заглушка вместо исходной геометрии
   и UV TC4.
4. **Outer Lands зависал на «Загрузка территории…».** Перед телепортацией
   сервер синхронно создавал квадрат 5×5, то есть до 25 комнат лабиринта. В
   одиночной игре это блокировало integrated server до завершения тяжёлой
   генерации, поэтому клиент оставался на экране загрузки без crash-report.
5. **Перегонный куб засорял чат.** Повторная диагностическая строка добавлялась
   в историю чата при каждом использовании.

## Внесённые исправления

### Жезлы, посохи и скипетры

- Удалены legacy-смещение `Y=1.50` и завышенный first-person масштаб.
- Добавлены отдельные ограниченные матрицы first-person и third-person.
- Оригинальная геометрия `ModelWand`, наконечники, стержни и UV сохранены.

### Node Transducer и стабилизаторы

- Мир и предмет преобразователя используют единый путь оригинальной модели
  `node_stabilizer.obj` с текстурами `node_converter.png` и
  `node_converter_over.png`.
- Кубическая item-заглушка заменена на `minecraft:builtin/entity` и отдельный
  custom item renderer.
- Принудительный `FULL_BRIGHT` удалён. Светящийся слой снова управляется
  диапазоном lightmap TC4, с непрозрачными вершинами на cutout render type.
- Обычный и улучшенный стабилизаторы продолжают использовать исходную модель,
  исходные UV и отдельную окраску overlay для advanced-варианта.

### Outer Lands

- До `ServerPlayer.teleportTo` создаётся только входная комната портала.
- Удалена синхронная генерация радиуса 2 (25 комнат).
- Загружается только конечный landing chunk, а две позиции выхода очищаются для
  безопасного появления игрока.
- Дальнейшее наполнение идёт center-first и ограничено максимум одной успешно
  созданной комнатой за проход раз в 40 тиков.

### Прочее

- Статус Alembic перенесён из постоянного чата в action bar.
- Runtime-манифест расширен с 82 до 88 сценариев.
- Все старые guards обновлены на версию 11.63.03 без ослабления проверок.

## Результаты проверок

- Полный статический набор: **67/67 PASS**.
- Целевой visual/Outer Lands guard: **37/37 PASS**.
- Liquid Death regression guard: **116/116 PASS**.
- Runtime manifest template: **88 тестов, PASS**.
- JSON, модели, ресурсы и Forge 1.19.2 API guards: **PASS**.

`compileJava` в контейнере не стартовал: Gradle Wrapper не смог разрешить
`services.gradle.org` (`UnknownHostException`). Доступна Java 21.0.10, тогда как
проект Forge 1.19.2 рассчитан на Java 17.

## Обязательная проверка в игре

Статические проверки не доказывают клиентскую parity. На Java 17 необходимо
проверить: обе руки и разные FOV для wand/staff/sceptre; GUI/ground/fixed/world
для трёх узловых устройств; вход в Outer Lands, бой с боссом, save/reload и
возврат через портал.
