# Отчёт о раунде портирования v11.64.11 — точная яркость Sun Scorned

Minecraft 1.19.2 · Forge 43.5.2 · целевая Java 17 · эталон Thaumcraft 4.2.3.5 (1.7.10)

## Найденное расхождение

Оригинальный `PotionSunScorned.func_76394_a` получает освещённость через `EntityLivingBase#getBrightness(1.0F)` (`target.func_70013_c(1.0F)`). Это значение берётся из таблицы яркости мира и не эквивалентно линейному делению сырого уровня света на 15.

В порте v11.64.10 использовалось:

```java
float brightness = target.level.getMaxLocalRawBrightness(pos) / 15.0F;
```

Из-за этого вероятностные зоны эффекта могли смещаться около исходных границ:

- горение только при `brightness > 0.5F`;
- лечение только при `brightness < 0.25F`;
- между 0.25 и 0.5 эффект не делает бросок RNG;
- яркая ветка требует открытого неба.

## Исправление production-кода

### `TC4WarpMobEffect.tickSunScorned`

- яркость теперь читается через `target.getLightLevelDependentMagicValue()` — современный эквивалент world/entity brightness-table sampling;
- удалены `getMaxLocalRawBrightness(pos)` и `/ 15.0F`;
- случайное число вызывается только после входа в яркую или тёмную ветку, поэтому нейтральная зона не изменяет RNG-последовательность;
- сохранены 4 секунды огня и лечение на `1.0F`.

### `TC4WarpRuntimeParity`

Добавлены source-linked константы и чистые предикаты:

- `SUN_SCORNED_BURN_BRIGHTNESS = 0.5F`;
- `SUN_SCORNED_HEAL_BRIGHTNESS = 0.25F`;
- `random * 30 < (brightness - 0.4) * 2`;
- `random > brightness * 2`;
- `sunScornedBurns(...)` и `sunScornedHeals(...)`.

## Проверки

- допустимый bright/zero-roll burn;

- обязательную видимость неба;
- допустимый dark/high-roll heal;
- строгую границу `< 0.25`, а не `<=`;
- отклонение равенства в heal roll (`random == brightness * 2`).
