# TC4 v11.63.55 — Essentia storage и Essentia Mirror runtime-контракты

## Цель этапа

Продолжить портирование Thaumcraft 4.2.3.5 после v11.63.54 и закрыть следующий доказуемый P1-пакет server-side механик essentia без объявления неподтверждённого runtime PASS.

## Что добавлено

В `TC4BlockEntityGameTests` добавлены четыре обязательных server-only GameTest.

### Filtered и Void Essentia Jars

Проверяется:

- фильтр допускает только выбранный аспект;
- обычная банка хранит максимум 64 единицы;
- пустая filtered jar имеет suction 64;
- полная обычная банка перестаёт втягивать essentia;
- filtered void jar потребляет overflow, визуально оставаясь на 64;
- полная filtered void jar сохраняет suction 48;
- аспект-фильтр и содержимое переживают NBT round-trip.

### Essentia Reservoir

Проверяется:

- доступ только со стороны `FACING`;
- смешанное хранение нескольких аспектов;
- точная вместимость 256;
- частичное принятие входа при достижении capacity;
- отсутствие suction после заполнения;
- сохранение facing и аспектов через NBT.

### Essentia Mirror: drain и rollback

Проверяется:

- взаимная связь пары зеркал;
- обнаружение удалённой банки;
- отказ от запроса больше одной единицы;
- извлечение ровно одной единицы;
- возврат в тот же выбранный источник при rollback.

### Essentia Mirror: направленный диапазон

Проверяется:

- источник позади рабочей стороны удалённого зеркала игнорируется;
- сканируется только передний объём `[0, 8)`;
- источник на дальнем допустимом смещении `cc=7` обнаруживается;
- reservoir на границе диапазона отдаёт одну единицу.

## Сборка и runtime

Подтверждённый JAR v11.63.55 не получен. Gradle Wrapper остановился до конфигурации проекта при загрузке Gradle 7.5.1:

```text
java.net.UnknownHostException: services.gradle.org
GRADLE_RC=1
```

В окружении обнаружен OpenJDK 21.0.10. Требуемый для проекта Java 17 runtime-прогон не выполнен. Поэтому 32 GameTest имеют статус `NOT_TESTED`.
