# Thaumcraft Legacy Rebuild v11.62.41

## Forge 1.19.2: Magical Leaves, ModelWand UV and original Research Table renderer

Целевая платформа:

- Minecraft 1.19.2;
- Minecraft Forge 43.5.2;
- Java 17;
- без NeoForge;
- без TerraBlender.

Этот этап продолжает исправление ошибок по присланным внутриигровым скриншотам. Механики и рендеры перенесены на API Forge 1.19.2, но размеры, UV, текстуры и поведение сверялись с исходниками Thaumcraft 4.2.3.5.

## 1. Исправлены листья Великого и Серебряного деревьев

На скриншоте крона Серебряного дерева превращалась в огромную полупрозрачную синюю решётку. Причина находилась не в самой PNG-текстуре: оба вида листвы были зарегистрированы как обычные `Block` с `noOcclusion()`. Современный рендер в таком случае показывал внутреннюю грань почти каждого соприкасающегося блока.

Сделано:

- добавлен `TC4MagicalLeavesBlock`, наследующий нативный `LeavesBlock` Forge/Minecraft 1.19.2;
- восстановлены состояния `distance`, `persistent` и `waterlogged`;
- восстановлена логика обновления расстояния до древесины и опадания листвы;
- соседние блоки одинаковой листвы снова скрывают внутренние грани;
- blockstate-файлы переведены на `multipart`, поэтому модель действует для всех комбинаций состояний листвы;
- сохранён `cutoutMipped`-рендер;
- Серебряная листва получила исходный уровень свечения 7 вместо почти незаметного 1;
- добавлены редкие магические частицы Серебряного дерева с частотой 1/500;
- восстановлены капли дождя под открытой листвой через современные частицы 1.19.2.

Основные файлы:

- `TC4MagicalLeavesBlock.java`;
- `ThaumcraftMod.java`;
- `blockstates/greatwood_leaves.json`;
- `blockstates/silverwood_leaves.json`.

## 2. Исправлен рендер жезлов

На скриншотах в руке иногда оставался только маленький элемент фокуса, а стержень и наконечники исчезали. Были обнаружены две отдельные причины.

### Неверный UV

Предыдущий адаптер накладывал всю текстуру 0..1 на каждую грань кубоида. Оригинальная модель TC4 использует логический атлас 64x32 и конкретные области:

- наконечники — UV `0,0`;
- стержень — UV `0,8`;
- фокус — отдельный куб 6x6x6.

Теперь добавлен адаптер оригинального `ModelRenderer/ModelBox`, который воспроизводит развёртку шести граней и размеры в пикселях модели.

### Смешанные смещения посоха

В TC4 существовали два разных смещения:

- внешнее смещение `ItemWandRenderer` на `+0.5Y`;
- внутреннее смещение `ModelWand` на `+0.2Y`.

Ранее они были объединены и частично компенсированы другими трансформациями, из-за чего стержень уходил из видимой области руки. Теперь оба этапа снова применяются отдельно, как в оригинале.

Дополнительно:

- стержни и наконечники используют `entityTranslucent`, чтобы прозрачные пиксели исходных текстур не отбрасывали весь элемент;
- сохранены отдельные масштабы обычного жезла, посоха и скипетра;
- сохранены нижние и двойные наконечники;
- сохранены оригинальные анимации WAVE/CHARGE и светящиеся стержни.

Основной файл: `WandItemRenderer.java`.

## 3. Исследовательский стол перенесён с оригинальной моделью TC4

Предыдущий JSON был только приближением. Он рисовал статичные коричневые кубоиды, не использовал настоящую модельную развёртку и не мог реагировать на содержимое двух слотов стола.

Добавлен `ResearchTableRenderer`, который переносит `ModelResearchTable` и `TileResearchTableRenderer` на `BlockEntityRenderer` Forge 1.19.2.

Перенесено:

- оригинальная текстура `restable.png` 128x64;
- оригинальная текстура свитка `restable2.png` с тем же логическим UV 128x64;
- единая столешница 32x4x16;
- четыре ножки и общая перекладина;
- исходный масштаб 1/16 блока;
- инверсия оси X и положение модели относительно верхней грани блока;
- четыре направления старых metadata 2..5;
- чернильница только при наличии настоящих Scribing Tools в слоте 0;
- перо из исходной `tablequill.png`;
- шесть слоёв пергамента с оригинальными смещениями, углами и масштабом;
- трубка свитка и окрашенная лента исследования только при записке в слоте 1;
- цвет ленты из NBT исследовательской записки.

Статическая world-модель теперь пуста, поэтому за BER больше не появляется второй коричневый стол. Для предмета в инвентаре сохранена отдельная модель `research_table_item.json`.

## 4. Восстановлено образование стола из двух обычных столов

Старый TC4 сохранял направление от активной половины к соседнему столу в metadata. В предыдущем порте направление терялось, и модель всегда смотрела в одну сторону.

Теперь:

- `ResearchTableBlock` хранит `HORIZONTAL_FACING`;
- `PRIMARY=true/false` сохраняет обе физические позиции исходных столов вместо удаления второй половины в воздух;
- только primary-половина создаёт BlockEntity, владеет инвентарём и рисует полную модель;
- нажатие по partner-половине перенаправляется к primary-половине и открывает тот же контейнер;
- разрушение любой половины корректно убирает вторую, но не создаёт двойной дроп;
- для обеих половин добавлены отдельные формы столешницы, ножек и перекладины вместо невидимого полного куба;
- направление устанавливается по реальному соседнему столу, использованному при преобразовании;
- поддерживаются вращение и зеркалирование blockstate;
- рендер использует исходную карту поворотов: NORTH 270°, SOUTH 90°, WEST 180°, EAST 0°;
- копия использованных Scribing Tools помещается в реальный слот 0;
- у игрока расходуется один предмет, кроме творческого режима;
- после установки инструмента BlockEntity сразу синхронизируется клиенту, поэтому чернильница и перо могут появиться без повторного открытия мира.

## 5. Защита от повторной поломки

Добавлен `tools/leaves_wand_table_guard.py`. Он останавливает GitHub Actions, если:

- магическая листва снова перестанет наследовать `LeavesBlock`;
- потеряются состояния листвы или уровень света Серебряного дерева;
- жезл снова потеряет UV `0,8`, оригинальный ModelBox-адаптер или два смещения посоха;
- Исследовательский стол снова получит статичный world-куб вместо BER;
- пропадут оригинальные текстуры, пергамент, перо или свиток;
- потеряется направление или `PRIMARY`-связь второй половины;
- Scribing Tools перестанут устанавливаться в слот;
- renderer перестанет регистрироваться на клиентской Forge-шине.

Также обновлён `runtime_visual_guard.py`: пустая JSON-модель Исследовательского стола теперь считается правильной только при наличии полного BER и отдельной inventory-модели.

## Статус сборки

Локальный JAR не создан. Gradle Wrapper остановился до конфигурации Forge-проекта:

```text
java.net.UnknownHostException: services.gradle.org
```

Поэтому этот архив содержит полный исходный Forge-проект, проверки и GitHub Actions. Компилируемый JAR должен быть получен из зелёного запуска GitHub Actions.

## Что обязательно проверить в собранном JAR

- крона Серебряного дерева больше не состоит из внутренних синих граней;
- листья сохраняются рядом с древесиной и опадают без неё;
- Серебряные листья дают мягкое освещение, а не превращают лес в прожектор;
- деревянный, Великодревесный и Серебродревесный жезлы видны полностью в GUI, первой и третьей руке;
- фокус расположен на наконечнике, а не отдельно под игроком;
- Исследовательский стол выглядит как один двухблочный объект во всех четырёх направлениях;
- после создания на столе видны чернильница и перо;
- записка в слоте отображает свиток с правильным цветом;
- открытие GUI и извлечение предметов не оставляют устаревший world-render;
- предыдущие исправления Таумометра, банок, матрицы, очков, стабилизатора и Magical Forest не регрессировали.
