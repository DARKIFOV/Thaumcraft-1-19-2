# Thaumcraft Legacy Rebuild 11.62.62 — продолжение порта строительных блоков, рецептов и JEI

## Итог

Версия **11.62.62** продолжает перенос обычных рецептов из `ConfigRecipes.java` TC4 4.2.3.5. В этой итерации добавлены или исправлены **40 исходных записей крафта**, а **26** предметов-зеркал заменены либо дополнены полноценными размещаемыми блоками.

Статический аудит версии: **PASS**. Полная Forge-компиляция не выполнена из-за недоступности сервера дистрибутивов Gradle; готовый JAR этой итерацией не заявляется.

## 1. Новые функциональные блоки

### Древесина

Добавлены настоящие блоки с ванильным поведением формы и размещения:

- ступени из Greatwood;
- ступени из Silverwood;
- плита из Greatwood;
- плита из Silverwood.

Для них добавлены blockstate, внутренние/внешние модели ступеней, модели верхней и нижней плиты, item-модели, loot table и исходные рецепты с выходом 4 ступеней или 6 плит.

### Хранилища и декоративные блоки

Под точными legacy-ID теперь зарегистрированы реальные `BlockItem`, а не плоские предметы-заглушки:

- `thaumcraft:tc4_block_thaumium`;
- `thaumcraft:tc4_block_tallow`;
- `thaumcraft:tc4_block_crystal_cluster`.

Также добавлены:

- `thaumcraft:amber_block`;
- `thaumcraft:amber_bricks`;
- `thaumcraft:item_grate`.

Старые ID исключены из цикла регистрации плоских исследовательских зеркал, чтобы один registry ID не получал две конкурирующие реализации.

### Item Grate

Добавлен отдельный `ItemGrateBlock`:

- состояние `open=true/false` заменяет исходные metadata-варианты решётки и закрытого люка;
- правый клик переключает состояние;
- в открытом состоянии выброшенные предметы проходят сквозь пластину;
- игроки и существа продолжают сталкиваться с пластиной;
- для двух состояний используются отдельные оригинальные текстуры и cutout-rendering.

Это статически проверенная аппроксимация поведения TC4. Коллизию предметов, звук и переключение состояния необходимо проверить в клиенте после полноценной сборки.

## 2. Полный набор сальных свечей

Добавлены **16** размещаемых вариантов:

- белая свеча;
- 15 окрашенных цветов Minecraft.

Для каждой свечи подготовлены:

- отдельный registry ID;
- blockstate, cross-модель и item-модель;
- анимированная вертикальная текстура `16×128`;
- `.png.mcmeta`;
- cutout render layer;
- собственный loot table;
- английская и русская локализация.

Рецепты:

- нить и два куска сала дают 3 белые свечи;
- белая свеча и соответствующий краситель дают окрашенный вариант;
- белый краситель и любая свеча из тега `thaumcraft:tallow_candles` возвращают белую свечу.

## 3. Исправленные и перенесённые обычные рецепты

### Строительные блоки

Перенесены:

- Greatwood/Silverwood stairs;
- Greatwood/Silverwood slabs;
- упаковка и распаковка таумиевого блока;
- упаковка и распаковка блока сала;
- Amber Block и Amber Bricks;
- обратное получение янтаря из обоих блоков;
- Item Grate.

### Кристальные скопления

Исправлены все шесть первичных скоплений:

- каждое теперь создаётся из **шести одинаковых осколков**;
- удалены шесть неоригинальных обратных рецептов «скопление → осколок»;
- сбалансированное скопление создаётся из всех шести первичных осколков;
- результат сбалансированного рецепта использует функциональный legacy-ID `tc4_block_crystal_cluster`.

### Essentia Phial, Table и Scribing Tools

- Essentia Phial восстановлен к исходной форме: один clay ball, три glass, выход **8**;
- удалена старая дублирующая копия рецепта фиала;
- Table принимает любые `minecraft:wooden_slabs` и `minecraft:planks`;
- добавлен исходный вариант Scribing Tools из пустого Essentia Phial, пера и чёрного красителя;
- альтернативные существующие варианты сохранены, но снабжены явными source-маркерами.

### Метаданные происхождения

Для чувствительных рецептов сохранено поле `tc4_source`, указывающее соответствующую запись `ConfigRecipes.initializeNormalRecipes`. Это позволяет отличить исходный перенос от совместимых современных альтернатив.

## 4. JEI

Все 40 рецептов этой итерации используют стандартные типы:

- `minecraft:crafting_shaped`;
- `minecraft:crafting_shapeless`.

Они должны поступать в обычную crafting-категорию JEI через стандартный менеджер рецептов, без второй ручной регистрации и без дублей с пользовательскими категориями.

Текущие количества в `data/thaumcraft/recipes`:

| Тип | Количество |
|---|---:|
| Shaped | 87 |
| Shapeless | 38 |
| Стандартных крафтов для обычной категории JEI | **125** |
| Vanilla smelting | 5 |
| Counted smelting | 4 |

Пользовательские категории Arcane Workbench, Alchemy и Infusion в `TC4JeiPlugin` сохранены.

## 7. Ограничение сборки и обязательный runtime-тест

Команда `./gradlew compileJava --no-daemon --stacktrace` остановилась до компиляции исходников: Gradle Wrapper не смог получить `gradle-7.5.1-bin.zip` из-за `UnknownHostException: services.gradle.org`.

Перед релизом обязательно проверить в клиенте:

1. компиляцию `StairBlock` supplier-конструктора и `EntityCollisionContext#getEntity()` на точной связке Forge 43.5.2;
2. все повороты, внутренние и внешние углы ступеней;
3. верхнюю/нижнюю/двойную плиту;
4. анимацию, прозрачность, освещение и предметную модель всех 16 свечей;
5. прохождение только `ItemEntity` через открытую Item Grate;
6. переключение и сохранение состояния Item Grate;
7. размещение и добычу exact-ID блоков в существующем мире;
8. отображение всех 125 стандартных рецептов в JEI без дублей;
9. фактический результат каждого из 40 рецептов этой итерации.
