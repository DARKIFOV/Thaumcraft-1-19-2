# Thaumcraft Legacy Rebuild 1.19.2 — v11.63.20

## Стадия

**Tallow Candle / Сальная свеча, 16 цветов** — восстановление координатно-зависимой геометрии `BlockCandleRenderer` из Thaumcraft 4.2.3.5 на Forge 1.19.2.

## Исходный контракт TC4

- Один блок использовал 16 metadata-цветов; в современном порте они сохранены как 16 отдельных registry ID.
- Тело свечи занимает `6/16..10/16` по X/Z и имеет высоту `8/16`.
- Фитиль занимает `0.475..0.525` по X/Z и высоту от `8/16` до `10/16`.
- В мире создаётся `new Random(x + y * z)`, затем от 1 до 5 восковых подтёков.
- Подтёки чередуют ось, случайно выбирают сторону, одну из двух позиций и высоту 1–3 пикселя.
- В inventory-render исходного TC4 подтёки отсутствуют: рисуются только тело и фитиль.
- Блок не имеет коллизии, требует опору снизу, испускает дым и пламя на высоте `y + 0.7` и стабилизирует infusion.

## Реализация Forge 1.19.2

- `TallowCandleBlock` переведён на `BaseEntityBlock` и `RenderShape.ENTITYBLOCK_ANIMATED`.
- Добавлен лёгкий `TallowCandleBlockEntity`, не хранящий игровой NBT и используемый только как источник позиции для рендера.
- Один `BlockEntityType` зарегистрирован для всех 16 цветовых вариантов.
- `TallowCandleRenderer` вручную строит шесть граней каждого cuboid через block atlas и `Sheets.cutoutBlockSheet()`.
- Формула seed и последовательность вызовов `Random` сохранены, включая Java int overflow.
- Текстура воска выбирается по registry path конкретного блока; фитиль использует общий `block/tc4/candlestub`.
- `TallowCandleBlockItem` подключает единый `TallowCandleItemRenderer` через `IClientItemExtensions`.
- Все 16 item-model JSON переведены на `builtin/entity`; предметный рендер вызывает общий код с флагом `world=false`, поэтому подтёки не появляются.
- Сохранены форма выделения, пустая collision shape, проверка опоры, удаление после потери опоры, свет 14, частицы и infusion-stabilizer контракт.

## Компиляция

`./gradlew compileJava --stacktrace --no-daemon` снова остановился до запуска Java-компилятора: Gradle Wrapper не смог разрешить `services.gradle.org` при загрузке Gradle 7.5.1 (`UnknownHostException`). В среде доступен OpenJDK 21.0.10; отдельного JDK 17 и локального Gradle 7.5.1 нет.

Поэтому JAR не создан. Реальные проверки atlas orientation, GUI/hand transforms, culling, shaders/resource packs, resource reload, save/reload, dedicated server и двух клиентов имеют статус `NOT_TESTED`.

## Известные адаптации

- В TC4 цвет задавался OpenGL tint поверх общей анимированной текстуры; порт использует уже подготовленные цветные atlas sprites для 16 registry-блоков.
- Для получения координат рендеру используется render-only BlockEntity. Это увеличивает число BlockEntity по сравнению с оригиналом, но не добавляет ticker, capabilities или сохраняемые поля.
- Фиксированное освещение граней старого `RenderBlocks` заменено современными cutout-вершинами с текущим packed light.
