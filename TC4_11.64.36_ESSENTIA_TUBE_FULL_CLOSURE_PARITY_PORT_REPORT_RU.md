## Исправления production-кода

- Минимальный suction всех tube-подтипов исправлен с неоригинального 1 на исходный **0**.
- Восстановлены исходные интервалы: suction раз в **2 тика**, transfer/equalization раз в **5 тиков**, обновление bellows у buffer раз в **20 тиков**.
- Обычная труба распространяет suction как `neighbour − 1`; Restricted Tube — как `neighbour / 2`.
- Filtered Tube принимает только типизированную Jar Label. Фиал больше не используется как неоригинальный способ настройки.
- Shift-click снимает фильтр и возвращает пустую Jar Label.
- One-way Tube применяет направление только к расчёту suction/equalization и не получает двойную блокировку сторон.
- Закрытый valve больше не удаляется из tube topology. Он подавляет suction, но позволяет соседней открытой стороне забрать уже находящуюся внутри единицу essentia.
- Возвращён ручной right-click toggle клапана, redstone `flow/hadpower`, звук squeek и клиентская анимация по 20 градусов до 360.
- Buffer хранит до **8** единиц смешанной essentia, обновляет bellows и использует choke 0/1/2: normal, blue suction 1, red suction 0.
- Choke доступен только buffer, а не всем трубам.
- Восстановлен comparator output buffer: 0 для пустого и 1…15 по заполнению.
- Side toggle теперь сразу обновляет baked connection properties и геометрию блока на клиенте.
- Возвращены исходные core/arm bounds, valve transform, buffer blue/red choke rods, venting и aspect/filter rendering.
- NBT разделён по исходным подтипам. Buffer сохраняет root `Aspects`, `open`, `choke`; обычные трубы — `type`, `amount`, `side`, `open`, `stype`, `samount`; filter добавляет `AspectFilter`, valve — `flow`/`hadpower`.
- Временные `venting`, `ventColor`, tick counters и bellows не записываются. Старые compounds порта читаются только как migration path.
- Проверены по SHA-256 **10** оригинальных tube/valve ресурсов.
